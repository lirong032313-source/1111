# AI 内容分发三件套 · 学员使用指南

> **一句话**：装好这 3 个 Skill，你的 AI 就能帮你发公众号、发小红书、发微信消息。
>
> **适用对象**：实操营学员 / 已有 WorkBuddy 或 OpenClaw 的用户
>
> **前置条件**：电脑已安装 WorkBuddy（或 OpenClaw），能正常跟 AI 对话

---

## 这三件套能做什么？

| 工具 | 能干什么 | 一句话触发 |
|------|---------|-----------|
| 公众号发布 | 写好的文章一键存入微信草稿箱 | 「帮我发到公众号草稿箱」 |
| 小红书运营 | 选题、写文案、发笔记、回评论 | 「帮我发一条小红书」 |
| 微信消息发送 | 给指定联系人/群聊发文字 | 「给xxx发微信：xxxx」 |

---

## 第一步：确认你有这几个 Skill

在 AI 对话框里，直接说：

> **「检查一下我有没有安装公众号发布和小红书的 skill」**

如果你的 AI 说"已经有了"，跳到第三步直接用。

如果说"没有"或找不到，继续第二步安装。

---

## 第二步：安装（只需做一次）

### 方式 A：让 AI 帮你装（推荐）

直接对 AI 说：

> **「帮我安装这三个 skill：wechat-article-publisher、xiaohongshu-ops、wechat-message-sender」**

AI 会自动执行安装命令，等它说"安装完成"就行。

### 方式 B：手动安装

如果 AI 装不了，复制以下命令到终端执行：

```bash
# 公众号发布
npx skills add cloudsen/eliteforge-skills@wechat-article-publisher -g -y

# 小红书运营（通常预装，如果没有）
npx clawhub install xiaohongshu-ops --workdir ~ --dir .workbuddy/skills

# 微信消息发送
npx skills add https://github.com/cloudsen/eliteforge-skills --skill wechat-message-sender -g -y
```

> ⚠️ **小红书额外要求**：需要有浏览器工具（openclaw browser），如果 AI 提示"浏览器未启用"，按它给的链接操作即可。

## 第三步：开始用！（自然语言交互模板）

下面是**你直接复制发给 AI 就能用的话术模板**——不需要记任何技术参数。

### 📮 公众号发布 → 存草稿箱

#### 场景1：把写好的文章发到公众号

```
帮我把这篇文章发到公众号草稿箱：
[粘贴文章内容或附上文件路径]
```

AI 会自动：
1. 把文章格式化（Markdown/HTML 都行）
2. 调用微信 API 存入草稿箱
3. 告诉你"已存好，去后台确认发布"

#### 场景2：从文件发布

```
把 /Users/xxx/我的文章.md 发到公众号草稿箱
```

#### 场景3：发"小绿书"格式（图文卡片风格）

```
把这篇文章以小绿书格式发到公众号草稿箱：
[内容]
```

> 💡 **重要规则**：AI 只会存草稿箱，不会替你点"发布"。你需要自己去微信公众号后台预览一下，确认没问题再手动发布。这是安全设计，防止误发。

#### 公众号前置配置（第一次用需要）

如果你还没配置过公众号，AI 会问你：

> "请提供你的公众号 AppID 和 AppSecret"

去 [微信公众平台](https://mp.weixin.qq.com) → 开发 → 基本配置 里找到这两个值，给 AI 就行。只需要配一次，之后每次说"发公众号"就能直接用。

---

### 📕 小红书运营 → 全链路

#### 场景1：发一篇图文笔记

```
帮我发一条小红书图文笔记：

标题：[你的标题，建议20字内]
正文：
[你的正文内容]
话题：#话题1 #话题2 #话题3
```

AI 会自动：
1. 打开小红书创作平台（内置浏览器）
2. 上传封面图 / 生成文字配图封面
3. 填入标题和正文
4. 添加话题标签
5. **停在"发布"按钮处等你确认**

> ⚠️ 同样是安全设计——AI 不会直接点"发布"，它会截图给你看效果，你说"可以发了"它才点。

#### 场景2：只写不发（先出内容）

```
帮我写一条小红书笔记，主题是[你的主题]，先不出
```

AI 会输出：
- 标题备选（3-5个）
- 正文初稿
- 封面文案建议
- 话题标签组合

你觉得满意了再说"发出去"。

#### 场景3：爆款复刻（Viral Copy）

```
这条小红书很火，帮我学习它的风格，写一条类似的内容：
[贴一个爆款笔记的链接]
```

AI 会分析原笔记的结构、标题套路、封面风格，然后仿写一条同款但不抄袭的新内容。

#### 场景4：选题 + 竞品分析

```
帮我分析小红书上[关键词]方向的热门内容，给我5个选题建议
```

AI 会去抓取小红书的推荐流数据，分析什么标题/角度/互动高，给你一份选题清单。

#### 场景5：评论回复

```
帮我看下小红书最新通知里的评论，挑几条重要的帮我回复
```

AI 会读通知页的评论，按你的账号人设风格写回复（每条都会让你过目后再发）。

### 💬 微信消息发送 → 私聊/群发

> ⚠️ 这个功能目前仅支持 **macOS + 微信桌面版**

#### 前置操作（一次性授权）

第一次使用前，需要在 Mac 上授权辅助功能权限：

1. 打开 **系统设置 → 隐私与安全性 → 辅助功能**
2. 点 **+** 号
3. 把你用的终端应用（Terminal / iTerm / VS Code 终端）加进去
4. 确保**微信桌面版已打开并登录**

#### 使用方式

```
给[联系人名字]发微信：[你想发的消息内容]
```

**示例：**

```
给 ClawBot 发微信：今天的AI情报简报已生成，请查收～
```

```
给产品研讨群发微信：明天的会议改到下午3点
```

> 💡 **关键点**：联系人名字必须和微信里显示的**完全一致**（包括空格和符号）。如果不确定，可以让 AI 先试跑不发送（dry-run）来验证能不能找到这个人。

#### 试跑模式（测试能不能定位到联系人）

```
给[联系人名字]发微信：测试消息（dry-run）
```

这样只会搜索联系人，不会真的发出消息。确认能找到后去掉 dry-run 再正式发。

## 常见问题

| 问题 | 解决方法 |
|------|---------|
| AI 说"找不到公众号 API Key" | 需要配置 AppID 和 AppSecret（见上文公众号前置配置） |
| 小红书提示"浏览器未启用" | 按 AI 给的链接去开启 openclaw browser 工具 |
| 微信消息发送报错"权限不足" | 检查系统设置的辅助功能权限是否加了终端 |
| 找不到联系人 | 确认名字和微信里显示的完全一致，先用 dry-run 测试 |
| 公众号存草稿箱失败 | 检查 IP 白名单是否添加了当前公网 IP |
| 小红书停在发布按钮不动 | 这是正常的！截图给你看，你说"发"它才点 |

## 三件套配合使用的完整工作流示例

假设你要把一篇文章同时分发到多个平台：

```
第一步：「帮我基于这个主题写一篇2000字的公众号文章」
       → AI 写好文章

第二步：「帮我把这篇文章发到公众号草稿箱」
       → AI 存入草稿箱 ✅

第三步：「把同一篇文章改成小红书风格，帮我发一条笔记」
       → AI 改写 + 发到小红书编辑页，等你确认 ✅

第四步：「给学员群发微信：今天新文章已发，欢迎大家阅读」
       → AI 发送微信消息 ✅
```

**全程你只需要说话，AI 负责所有操作。**

## 进阶：做成自动化定时任务

如果你想每天自动执行某些操作，对 AI 说：

> **「每天早上10点自动生成一份AI情报简报，然后发到我的微信」**

或者：

> **「每周二和周四晚上8点提醒我：该发小红书了」**

AI 会帮你创建自动化任务，到时间自动执行。
