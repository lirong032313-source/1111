P03 · 核心概念

AI 助手 vs 数字员工

维度AI 助手数字员工
触发方式你问 → 它答自己有工作清单
任务范围单次对话多步工作流
责任输出一段文字完成一项交付
类比智能搜索远程同事
你的角色提问者主管/审核

金句：AI 助手是反应式的——你不问它它不动。数字员工是主动式的——它有自己的工作清单、SOP、KPI。

P05 · 核心区别

Skill vs 数字员工

维度Skill（技能）数字员工
定义单点任务模板完整工作单元（感知→决策→执行）
组成1个 promptWiki + Skill + 工作流
触发手动调用自动/半自动
能力单一一件事连续做多件事
依赖独立存在依赖知识库

类比：

Skill = 一把锤子
数字员工 = 一个木匠（知道什么时候用锤、钉什么、怎么组合）

前期用 Skill 练手感，等你发现"这个流程我老是在重复"，就升级成数字员工让它自动跑。

P08 · 加餐

Skill 创建加餐

Skill 听起来高大上，本质就是——你给 AI 一份说明书。

最简 Skill 结构：

my-skill/
├── SKILL.md        ← 触发词 + 说明
└── prompts/
    └── main.md     ← 核心提示词

金句：Skill 不是写代码，是写说明书。
一个 Skill 不到 30 行。
