## Part 1｜认知：什么是"数字员工"？（20 分钟）

### 🤖 1.1 数字员工 ≠ AI 助手（5 分钟）

**话术**：
> "我们先把概念讲清楚。
> 
> - **AI 助手**：你问它答（Query 模式）
> - **数字员工**：它代表你工作（Agentic 模式）
>
> 区别是什么？AI 助手是**反应式**的——你不问它它不动。数字员工是**主动式**的——它有自己的工作清单、SOP、KPI。"

**PPT P14：AI 助手 vs 数字员工对比表**

| 维度 | AI 助手 | 数字员工 |
|---|---|---|
| 触发方式 | 你问 → 它答 | 自己有工作清单 |
| 任务范围 | 单次对话 | 多步工作流 |
| 责任 | 输出一段文字 | 完成一项交付 |
| 类比 | 智能搜索 | 远程同事 |
| 你的角色 | 提问者 | 主管/审核 |

### 🪁 2.2 现场演示：从一句话到双平台稿（30 分钟）

> **演示原则**：用蓉蓉子真实当下的素材跑，不要用准备好的 demo 数据——"假"立刻被识破。

#### 步骤 ❶-❷｜碎片 → 选题（5 分钟）

**蓉蓉子操作**：
- 在微信发给龙虾："今天和木瓜熊聊咨询，发现她最大的痛点是不知道自己擅长什么。这其实跟我去年很像。"
- 等龙虾自动入库（学员已熟悉）
- 发指令："基于本周 Wiki 增量 + M1/M2/M3 母题，给我出本周 5 个选题"

**龙虾输出**（示例）：
```
📋 本周选题池（基于 Wiki + 你最近的素材）

1️⃣【M1·成长日记】
   "30 岁还不知道自己擅长什么"——我也曾是
   
2️⃣【M2·小而美产品】
   从 1v1 咨询里看到的 3 个用户痛点

3️⃣【M3·AI智囊团】
   我的小龙虾今天帮我做了什么

4️⃣【M1×M3】
   AI 不能替你想清楚定位，但能让你少走 3 年弯路

5️⃣【M2】
   为什么我从不让学员先做 SWOT
```

**话术**：
> "看到了吗？AI 不是凭空想选题，它**调用了我们昨天的木瓜熊咨询素材 + M1 故事母题 + 05 房间的痛点库**。这就是『底座』的力量。"

---

#### 步骤 ❸-❹｜选定主题 → GAP 框架大纲（8 分钟）

**操作**：
- 蓉蓉子选第 1 个选题
- 发指令："用 GAP 框架给这个选题出大纲。Gap=问题；Analysis=拆解；Principle=原则；Action=行动"

**龙虾输出**（示例）：
```markdown
# 30 岁还不知道自己擅长什么——我也曾是

## Gap（问题）
- 30 岁了，依然不知道自己擅长什么
- 试过很多领域，但没一个让你说"这就是我"
- 焦虑 + 自我怀疑

## Analysis（拆解）
- 误区 1：擅长 = 喜欢？不一定
- 误区 2：擅长 = 简历上写得最长的那段？不是
- 真相：擅长是"做这件事时你最不费劲，且持续被认可"

## Principle（原则）
1. 把"擅长"从"评估"变成"实验"
2. 用 3 个月小步快跑，不要 3 年想清楚
3. 看反馈而不是看自我感觉

## Action（行动）
1. 列出"过去 3 年别人主动找你帮忙的场景"
2. 在小范围测试 1 个微产品（199 元就行）
3. 看真实付费数据

## 调用引用
- 木瓜熊咨询（raw/咨询逐字稿/2026-04-29）
- 我自己的 1.0/2.0 故事（wiki/01_我是谁）
- 小而美产品方法论（wiki/02_我会什么）
```

#### 步骤 ❺｜初稿生成（7 分钟）

**操作**：
- 发指令："用 GAP 大纲 + 03 风格 + 01 身份生成初稿，控制在 1500 字"

**演示要点**：
- 现场让学员看到 AI 调用了 03 风格采样
- 输出的语气和金句是否像蓉蓉子
- 标注 [来源：xxx] 是否清晰

**🎤 金句**：
> "**这一段写出来，你的写作时间从 3 小时压缩到 30 分钟——但**前提是你的底座搭好了**。没底座，AI 写出来的就是『正确的废话』。"

#### 步骤 ❻｜四维润色检查表（5 分钟）

**PPT P18：四维润色检查表**

```
☐ 检查 1：语气像我吗？
   → 对照 03_我说过什么，读出来看看
   
☐ 检查 2：专业准确吗？
   → 对照 02_我会什么，有没有外行话
   
☐ 检查 3：用户视角吗？
   → 对照 05_我面对谁，能不能戳中读者
   
☐ 检查 4：价值观一致吗？
   → 对照 06_我怎么想，有没有越线
```

**演示**：
- 蓉蓉子现场用四维表过一遍 AI 初稿
- 标出 2-3 处需要"加人味"的地方
- 当场修改完成

**话术**：
> "记住——**AI 写 80%，你改 20%。这 20% 不能省，因为这 20% 才是『为什么是你』的部分。**"

## Part 3｜其他场景数字员工怎么打造（20 分钟）

### 🎯 3.1 数字员工的"通用打造方法"（8 分钟）

**话术**：
> "今天教你做了内容数字员工。但你的工作里还有很多其他场景对吧？教练答疑、咨询交付、社群运营……都可以做成数字员工。怎么做？"

**PPT P19：数字员工打造 5 步法**

```
Step 1：拆 SOP
   → 把这个场景的工作拆成 5-8 步
   
Step 2：每一步的输入/输出是什么？
   → 入参（什么触发）、出参（什么交付）
   
Step 3：每一步要调用什么知识？
   → 对应到六间房的哪个房间
   
Step 4：每一步需要什么工具？
   → 已有 Skill / 要新做一个 / 调外部 API
   
Step 5：把这 5 步写成一个 Skill
   → 用 skill-creator 工具半小时搞定
```

**举例**：把"教练答疑"做成数字员工
- 拆 SOP：收问题→分类→读历史相似案例→给出建议→记录
- 调用知识：05 用户痛点 + 02 方法论 + 06 决策框架
- 工具：wiki-query + 历史案例检索 + 微信回复

### 🛠 3.2 Skill 创建（轻量版加餐 12 分钟）

> 不展开所有方法，重点讲"最简的 Prompt Skill"——80% 学员能直接用上的。

**话术**：
> "Skill 听起来高大上，本质就是——**你给 AI 一份说明书**。说明书 = 一个 .md 文件。"

**最简 Skill 结构**：
```
my-skill/
├── SKILL.md          ← 触发词 + 说明
└── prompts/
    └── main.md       ← 核心提示词
```

**SKILL.md 内容示例**：
```markdown
# 行业周报生成器

## 触发词
"帮我出本周行业周报"
"周报"

## 说明
基于我的 Wiki + 本周 raw/articles/ 自动生成行业周报

## 调用流程
读 prompts/main.md → 执行
```

**prompts/main.md 内容**：
```markdown
# Role
你是 [我的名字] 的行业研究助手

# Task
扫描 raw/articles/ 本周文章
按 M1/M2/M3 母题分类
每个母题选 3 篇最有价值的
每篇用 100 字总结 + 给我 1 个行动建议

# Output Format
[周报模板]
```

**话术**：
> "看到了吗？**一个 Skill 不到 30 行**。你把它放在 my-wiki/skills/ 下，Cowork/WorkBuddy 自动识别。"

**🎤 金句**：
> "**Skill 不是写代码，是写说明书。**会写文档你就会写 Skill。"

### ❓ 4.2 Q&A（4 分钟）

**预备 Top 3 高频问题**：

1. **Q：我没有公众号，要不要发？**
   A：不用强求。这 21 天的目标不是"必须发布"，而是"跑通能力"。能跑通，未来你想发再发。

2. **Q：AI 写的我感觉读者会看出来？**
   A：80% 由 AI 写、20% 你改——改的是金句、故事、个人化细节。读者能看出 AI 的是"全 AI 写"，看不出"AI+你"。

3. **Q：数字员工跑久了会不会"漂移"，越来越不像我？**
   A：会。所以每周末做一次 Lint 体检（直播 1 教过），一个月调一次 CLAUDE.md。
