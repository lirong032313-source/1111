## 📐 认知锚点：LLM Wiki × 六间房

### 为什么以前的知识库搭不起来？

> 旧范式：**资料堆积** → 新范式：**让 AI 读懂你的大脑**

OpenAI 前研究负责人 Andrej Karpathy 今年提出了 **LLM Wiki** 范式：

> **传统 RAG 是"每次让 AI 临时去图书馆翻书"；**
> **LLM Wiki 是"你雇了一个私人图书管理员，他每天帮你整理书架"。**

### 三层架构（直播核心认知）

```
┌──────────────────────────────────────────┐
│  📐 Schema 层（规则/契约）                │ ← 你和 AI 共同维护
│  ─────────────────────────────           │
│  CLAUDE.md · 写作风格 · 边界 · 母题      │
│  → 告诉 AI "怎么像你"                     │
├──────────────────────────────────────────┤
│  📚 Wiki 层（六间房）                     │ ← AI 写、你读 + 纠错
│  ─────────────────────────────           │
│  01我是谁 02我会什么 03我说过什么          │
│  04我卖什么 05我面对谁 06我怎么想          │
│  → AI 自动维护的信息卡                    │
├──────────────────────────────────────────┤
│  📥 Raw Sources 层                       │ ← 你写、AI 只读
│  ─────────────────────────────           │
│  文章 / 对话记录 / 灵感碎片 / 课件 PDF    │
│  → 你的原始档案，不可变                   │
└──────────────────────────────────────────┘
```

### 金句（直播会重复 3 次）

> **「六间房解决"装什么"，LLM Wiki 解决"怎么活"。CLAUDE.md 是你和 AI 之间的契约。」**

### RAG vs LLMWiki 对比

| 维度 | 传统 RAG（你现在的用法） | LLM Wiki（这堂课教的） |
|---|---|---|
| 知识处理时机 | 查询时处理（每次重复算） | 摄入时处理（一次性编译） |
| 交叉引用 | 临时发现 | 预先构建并持续维护 |
| 矛盾检测 | 容易被忽略 | 摄入时主动标记 |
| 知识积累 | 无——每次从头开始 | 复利增长 |
| 一句话类比 | 每次去图书馆翻书 | 有一个私人图书管理员 |

## 🤝 搭子组合机制 —— 为什么这么搭配？

> 这是这次实操营区别于其他课程的核心设计之一。**知识库这件事，一个人很难坚持下来，但有搭子就不一样了。**

### 为什么需要搭子？

```
一个人的知识库之路：
  Day 1  兴奋搭建 🎉
  Day 3  遇到卡点 😵
  Day 7  "明天再说" 😴
  Day 14 又一个废弃的文件夹 🪦

有搭子的知识库之路：
  Day 1  一起搭建 🎉
  Day 3  互相帮忙解决卡点 🤝
  Day 7  搭子催你打卡 ⏰
  Day 14  两个人都有 MVP 了 🚀
```

### 搭子怎么配？

| 配对原则 | 说明 |
|---|---|
| **强弱混搭** | 有技术基础的 + 小白互补，互相成就 |
| **2-3 人一组** | 不宜太多，保持高效沟通 |
| **背景多元** | 不同行业/职能碰撞出更多灵感 |

### 搭子做什么？

| 动作 | 频率 | 说明 |
|---|---|---|
| 互打卡 | 每周 ≥ 2 次 | 在小群里发进度截图 |
| 先问搭子 | 遇到问题时 | 搭子答不出再问主群 |
| 互查作业 | 每周 1 次 | 按验收标准帮对方检查 |
| 技术互助 | 随时 | Obsidian/Claude/AI 操作问题 |

### 搭子激励机制

| 奖项 | 奖品 | 评选标准 |
|---|---|---|
| 🏆 **最佳搭子奖** | 私教抵扣 2000×2 + 小群深度辅导 + 学费全额退还 | 两人都高质量完成 21 天任务 + 互打卡最积极 |
| 🌟 **互助之星** | 私教抵扣 1000 + 学费退还 + 30min 999 咨询诊断 | 在主群帮助他人次数最多 + 被感谢最多 |

### 为什么这样设计？

> **三个底层逻辑**：
>
> 1. **社会承诺效应**：公开承诺 + 有人看着你，完成率提升 65%+
> 2. **技能互补效应**：技术强的帮弱的，弱的帮强的一起理清思路（知识内容层面），双向成长
> 3. **情感支持效应**：搭知识库是孤独的事，有同伴会让"放弃"的成本变高
>
> 往期数据：有搭子的学员完成率 **89%**，没有搭子的 **41%**。差了一倍不止。

### 分工原则

> **21 天里，AI 帮你写 80% 的卡，你写"非你不可"的 20%。这才是 AI 的正确打开方式。**

| 类型 | 内容 | 谁来做 | 数量 |
|---|---|---|---|
| ✋ **地基卡** | 一句话定位 / 核心价值观 / 写作风格 | 必须你自己写 | 约 3 张 |
| 🤖 **装修卡** | 方法论 / 金句库 / 用户画像 / 产品说明 | 丢素材给 AI | 约 80% |
