## 🔑 核心概念（先讲清楚3个词）

### 1. 什么是"微信打通ob"

不是技术黑话。意思是：

| 你的微信 | 对应 ob 的哪个位置 |
|---|---|
| 发给AI的灵感碎片 | → `00_Inbox`（临时候转） |
| 朋友圈的感悟 | → `3号房-金句库` 或 `6号房-思考框架` |
| 学员咨询的逐字稿 | → `01_Raw` → 提炼后 → `02_Wiki` |
| "帮我写篇公众号" | → AI调用知识库 → 生成文章 → `04_Output` |

**一句话：微信是入口，ob是仓库，AI是搬运工。**

### 2. 为什么需要"打通"

```
没有打通：
  微信(碎片) ←→ obsidian(孤岛) ←→ AI(陌生人)
  三者各干各的，你手动搬运，累且容易丢

打通后：
  微信 ←→ AI(中枢) ←→ obsidian(底座)
  你只负责"输入"和"确认"，中间全自动
```

### 3. 需要哪些工具

| 工具 | 角色 | 是否免费 |
|---|---|---|
| **微信** | 你的日常入口，随时随地能用 | ✅ 已有 |
| **OpenClaw / ClawBot** | 让AI能收到你在微信发的消息 | ✅ 免费 |
| **WorkBuddy** | AI的"办公室"，处理任务的地方 | ✅ 免费 |
| **Obsidian** | 本地知识库，你的数字大脑 | ✅ 免费版够用 |

> 💡 不需要买任何付费工具，这套方案全部免费可搭建。

## ❌ 常见坑点（直播必讲）

### 坑1：CLAUDE.md写得像简历，不像"你"

| ❌ 错误写法 | ✅ 正确写法 |
|---|---|
| "我是一个专业的知识管理咨询师，拥有5年经验..." | "我从2024年开始折腾知识库，踩了100个坑才摸出门道..." |
| "擅长：项目管理、团队协作、数据分析" | "我最拿手的事：把一团乱麻的信息，整理成AI能读懂的结构" |
| "风格要求：专业、简洁、逻辑清晰" | "写东西像跟朋友聊天，偶尔毒舌，经常自嘲" |

### 坑2：什么都往知识库里塞

```
不该存的：
  × 转发的文章链接（那是收藏夹的事）
  × 临时待办（那是日历/备忘录的事）
  × 别人的观点（除非你加了评论）

该存的：
  ✓ 你自己的原创感悟
  ✓ 你说过的话（金句/语录）
  ✓ 你的方法论和框架
  ✓ 用户真实的反馈和提问
```

### 坑3：只存不调，知识库变成了死库

> **检验标准：你上次从知识库里调取内容是什么时候？如果超过一周没调用过，你的库已经"腐烂"了。**

解法：养成一个习惯——每次写内容前，先让AI"从我的知识库里找找相关的"。

### 坑4：期望一键完美

```
第一次入库 → 可能分类不准 → 手动调整
第一篇文章 → 可能风格不像 → 标注润色点 → 第二篇就好多了
第一个月 → 可能在搭建期 → 坚持过21天就会看到质变
```

> ⚡ **注意：Day 1-7 的前半部分（搭目录、写 CLAUDE.md）已在直播①完成**
> 
> **本节课后的作业从"接通测试 + 跑通循环"开始**

### 第1周作业：接通 & 养成习惯

| 天数 | 任务 | 预计时间 |
|---|---|---|
| Day 1 ✅ 已做 | 建好 ob 目录结构 + 写 CLAUDE.md v1 | 直播①已完成 |
| Day 2 ✅ 已做 | 往每个房间放入至少 1 份内容 | 直播①已完成 |
| **Day 3**（今天！）| **确认 CLAUDE.md 有知识库路径 + 发第 1 条测试消息** | **10min** |
| Day 4 | 发 3 条不同类型的碎片（金句 / 方法论 / 故事各1条），看 AI 分类准不准 | 15min |
| Day 5 | 让 AI 从知识库里调内容，写一段 200 字短文 | 15min |
| Day 6 | 回顾入库结果：哪些分类对了？错的调整 CLAUDE.md 规则 | 15min |
| Day 7 | 周复盘：这周存了多少？调用了几次？给知识库打个分（1-10）| 10min |

### 第2周作业：跑通内容生产

| 天数 | 任务 |
|---|---|
| Day 8 | 用知识库产出一篇完整文章："帮我用我的风格写篇关于 xx 的公众号" |
| Day 9 | 复盘 Day8 文章：哪些地方像你？哪些不像？更新 CLAUDE.md |
| Day 10 | 练习"朋友圈扩写"：找一条旧朋友圈扩成公众号 |
| Day 11 | 做一次知识库体检：哪些房间空的？哪些内容重复？ |
| Day 12 | 补充最空的那个房间，至少加 3 张信息卡 |
| Day 13 | 用知识库内容写一篇小红书 |
| Day 14 | 周复盘：这周产出了什么？知识库有什么变化？ |

### 第3周作业：形成习惯 & 迭代优化

| 天数 | 任务 |
|---|---|
| Day 15 | 设定固定节奏：每天什么时候存碎片？什么时候写内容？ |
| Day 16-20 | 每天至少 1 次"存"+ 1 次"调"，保持流水运转 |
| Day 21 | 最终复盘：对比 Day1 和今天，你的知识库和工作流发生了什么变化？ |
