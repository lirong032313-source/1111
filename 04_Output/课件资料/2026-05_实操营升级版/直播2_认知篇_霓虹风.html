为什么分三层？

工程上的「关注点分离」

打通前（直播1之后你的状态）

微信(碎片) ←→ Obsidian(孤岛) ←→ AI(陌生人)

三者各干各的，你手动搬运，累且容易断

打通后（这节课之后）

微信 ←→ ClawBot(信使) ←→ WorkBuddy(助理) ←→ Obsidian(底座)

你只负责「发微信」和「确认」，中间全自动

类比：IT 部门会让电脑上同时运行「打印机+邮件+文件柜+ChatGPT」在同一个软件里吗？
每个工具做自己最擅长的事，组合起来才稳定。

核心区别

这是最重要的认知升级：

🤖 大部分AI工具

ChatGPT / Claude / 豆包 / DeepSeek

帮你「想」

  • 回答问题
  • 写文章初稿
  • 给建议、出方案

但：说完就完了，不会帮你执行

🦞 ClawBot + WorkBuddy

基于OpenClaw底座

帮你「干」

  • 收到消息 → 自动入库到Obsidian
  • 识别意图 → 调用Skills执行工作流
  • 定时任务 → 自动推送日报/汇总
  • 写完文件 → 帮你更新到对应房间

干完才告诉你结果

别人用AI是『想完自己干』，
你用ClawBot是『说一声它就干了』。

WorkBuddy 和 Claude Code 的关系

不是一个，是两个

🐾

WorkBuddy

像：24小时值班的主管

  • 定时任务自动跑（每天9点发日报）
  • 微信消息自动响应
  • 调度多个Skills工作流
  • 你不在线，它也在后台跑
VS
💻

Claude Code

像：随时听你话的桌面秘书

  • 你在终端/桌面直接说「帮我整理文件夹」
  • 直接读写本地文件、改代码
  • 每次主动启动，你说什么它做什么

Claude Code 是你的桌面贴身秘书，WorkBuddy 是你的24小时数字员工。
两个都用，互为补充。
