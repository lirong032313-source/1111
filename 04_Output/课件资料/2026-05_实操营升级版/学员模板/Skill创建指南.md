# Skill 创建指南（轻量版）

> 直播 3 物料 · 学员使用版
> **定位**：让你在 15 分钟内创建一个可用的 Skill

---

## Skill 是什么？

Skill = 一个可复用的 AI 任务模板。

当你发现某个任务需要重复做 3 次以上，就该把它做成一个 Skill。

---

## Skill 的基本结构

```
# Skill 名称

> 一句话说明这个 Skill 做什么

## 触发词
- 触发词1
- 触发词2

## 角色定义
| 项目 | 内容 |
|------|------|
| 名字 | xxx |
| 擅长 | xxx |
| 不擅长 | xxx |

## 处理流程
1. 读取用户输入
2. 调用 Wiki 资源（如果有）
3. 执行任务
4. 返回结果

## 注意事项
- xxx
```

---

## 5 分钟快速创建法

### Step 1：确定这个 Skill 要解决什么问题

写下来：
```
当用户说「_______」时，我要帮他_______。
```

### Step 2：列出触发词（2-5 个）

触发词 = 用户说什么时你要出动。

```
举例：
触发词：「写公众号」「生成文章」「公众号草稿」
```

### Step 3：定义角色（3 句话）

```
你是一个_____，你的任务是_____。
你的风格是_____。
```

### Step 4：写处理流程（3-5 步）

```
1. 读取用户的主题/素材
2. 调用 Wiki 中的 [相关资源]
3. 按 [结构] 生成内容
4. 自检并标注需优化处
```

### Step 5：列出来源（Wiki 路径）

```
- 风格采样：02_Wiki/3号房-我说过什么/风格采样/
- 金句库：02_Wiki/3号房-我说过什么/金句库/
- CLAUDE.md（写作风格总纲）
```

## 案例：公众号草稿生成 Skill

```yaml
# 公众号草稿生成 Skill

> 根据选题/素材生成符合蓉蓉子风格的公众号文章草稿

## 触发词
- 写公众号
- 生成公众号文章
- 公众号草稿

## 角色定义

| 项目 | 内容 |
|------|------|
| **名字** | 蓉蓉子的公众号创作助理 |
| **擅长** | 模仿蓉蓉子的写作风格创作文章 |
| **风格** | 温暖、有能量、像朋友聊天 |

## 处理流程

### Step 1：确认选题
用户给方向 → 出 3-5 个细分选题供选择
用户给主题 → 直接进入 Step 2

### Step 2：调取素材
从 Wiki 调用：
- 风格采样（学习语气）
- 金句库（引用金句）
- 人设故事（身份背景）

### Step 3：生成标题
直接生成 3 个 15-20 字的爆款标题供选择

### Step 4：生成正文
按结构生成约 2000-3000 字的深度内容

### Step 5：自检润色
检查：
- [ ] 语气像吗？
- [ ] 有没有人格化的声音？
- [ ] 禁忌词都避开了？

## 调用的 Wiki 资源
- 02_Wiki/3号房-我说过什么/风格采样/
- 02_Wiki/3号房-我说过什么/金句库/
- 02_Wiki/1号房-我是谁/（人设故事）
- CLAUDE.md

## 注意事项
- 不要编造故事，必须真实
- 有需要用户亲自润色的地方用「⚠️」标注
```

---

## 创建后的测试清单

创建完一个 Skill 后，用这些问题测试它：

- [ ] 触发词能覆盖主要场景吗？
- [ ] 角色定义清晰吗？
- [ ] 流程走完大概需要几步？
- [ ] Wiki 资源都找得到吗？
- [ ] 输出结果符合预期吗？

---

## 常见错误

❌ **触发词太少** → 只有一种说法能触发，用户不知道怎么说

✅ **触发词 3-5 个** → 覆盖主要说法，如「写公众号」「生成文章」「公众号草稿」

❌ **流程太长** → 超过 7 步容易出错

✅ **流程 3-5 步** → 每步聚焦一个动作

❌ **不写 Wiki 来源** → AI 不知道去哪里找素材

✅ **列出来源路径** → AI 能精准调用

---

## 下一步

创建完 Skill 后：

1. **测试 3 次** → 每次记录哪里可以优化
2. **迭代** → 根据测试结果调整流程
3. **固化** → 跑通后写成文档，存在 `05_System/skills/[你的Skill名]/`
