## Part 1｜认知：三个工具，三个角色（20 分钟）

> **教学策略**：很多学员会混淆"Cowork、WorkBuddy、ClawBot 到底是什么"。这部分必须用**职责类比**讲清楚，否则后面安装会一头雾水。

### 🎯 1.1 一张图说清三个工具的关系（8 分钟）

**PPT P10：工具三角图**

```
         ┌─────────────────────────┐
         │   你（用户）             │
         │   微信里发消息           │
         └──────────┬──────────────┘
                    ↓
         ┌──────────────────────────────┐
         │  🦞 微信 ClawBot              │
         │  （桥梁 = 信使）              │
         │  把你的微信消息转给后台        │
         │  把后台的回应送回你的微信      │
         └──────────┬───────────────────┘
                    ↓
         ┌──────────────────────────────┐
         │  🐾 WorkBuddy / Cowork        │
         │  （自动化主脑 = 助理大脑）    │
         │  ─ 理解你说的是什么意图        │
         │  ─ 决定调用哪个 Skill         │
         │  ─ 调度工作流执行              │
         └──────────┬───────────────────┘
                    ↓
         ┌──────────────────────────────┐
         │  📚 Obsidian + my-wiki         │
         │  （知识库 = 你的大脑文件库）   │
         │  ─ raw/ 你的素材               │
         │  ─ wiki/ AI 整理后的信息卡     │
         │  ─ CLAUDE.md 规则              │
         └──────────────────────────────┘
```

**金句话术**：
> "**Obsidian 是大脑（装知识），WorkBuddy 是助理（处理任务），ClawBot 是信使（送消息）**——三个工具各司其职，不是相互替代的关系。"

---

### 🪴 1.2 为什么要分这三层？（5 分钟）

**话术**：
> "你可能问：为什么不能一个工具搞定？我反过来问你——
> 
> 你公司的 IT 部门会让你的电脑同时运行『打印机+邮件+文件柜+ChatGPT』在同一个软件吗？不会。
>
> **每个工具做自己最擅长的事，组合起来才稳定。** 这就是工程上的『关注点分离』。"

**PPT P11：分工对比表**

| 工具 | 干什么 | 不干什么 |
|---|---|---|
| **Obsidian** | 看/编辑/搜索本地 .md 文件 | 不发微信、不调 AI、不跑工作流 |
| **WorkBuddy / Cowork** | 跑工作流、调用 Skills、读写文件、调用 AI | 不负责消息收发 |
| **微信 ClawBot** | 把微信消息和后台连起来 | 不做任何 AI 计算 |

---

### 🌱 1.3 Cowork 和 WorkBuddy 的差异（重点 7 分钟）

> **学员高频疑问**：Claude Cowork 和 WorkBuddy 都是 AI 自动化工具，到底用哪个？

**话术**：

> "我把这两个比作『两个员工』：
> - **Claude Cowork**：像一个**贴身秘书**——你在桌面上跟它说『把这个文件整理一下』，它现场处理
> - **WorkBuddy**：像一个**24小时值班的主管**——你不在线它也在后台跑，定时任务、自动化工作流、复杂调度
>
> **怎么选？**
> - 你**桌面上随手做的**事 → Cowork（每天即时处理）
> - 你**希望它后台自己跑**的事（比如每天 9 点抓 AI 资讯）→ WorkBuddy
>
> **这次实操营我们两个都用**：Cowork 处理你日常的本地操作，WorkBuddy 跑龙虾的后台工作流。"

**PPT P12：Cowork vs WorkBuddy 选择决策树**

```
你想做什么？
    │
    ├─ 在桌面随手让 AI 帮我整理文件、改文档、跑代码
    │   → Claude Cowork（即时、本地、每次主动启动）
    │
    ├─ 我发条微信给 AI，它自动帮我入库、回复
    │   → WorkBuddy + ClawBot（24小时在线、自动触发）
    │
    └─ 定时任务（每天 9 点 AI 日报、每周一周报）
        → WorkBuddy 的 Automation 模块
```

### 🛠 2.2 安装 ClawBot 三步走（12 分钟）

**步骤 1：注册 / 登录 WorkBuddy**（4 分钟）

### 📦 2.3 完成龙虾的"养虾配置"（15 分钟）

**配置 1：让龙虾认识你的 my-wiki**（5 分钟）

```
打开 WorkBuddy 设置 → 知识库绑定 →
选择本地文件夹 → my-wiki 路径 →

```

✅ 验证：龙虾能复述你 CLAUDE.md 的内容 = 接入成功

**配置 2：让龙虾学会 Ingest 工作流**（5 分钟）

在 WorkBuddy 添加 Automation 规则：

```yaml
触发条件：微信收到 [你] 发送的消息
判断意图：是否是"素材入库"类型
  - 包含链接 → 抓取全文
  - 包含图片 → OCR 提取文字
  - 纯文字想法 → 直接入库
执行动作：
  1. 按 CLAUDE.md 规则分类到 raw/对应类型/
  2. 提炼信息卡 → wiki/对应房间/
  3. 更新 index.md 和 log.md
  4. 回复："✅ 已入库 [房间]"
```

**演示**：蓉蓉子现场发一条想法 → 看龙虾完整跑完入库流程 → 学员同步操作。

**配置 3：让龙虾能 Query**（5 分钟）

### 📐 3.2 Schema 深化：让 AI 越来越懂你（15 分钟）

> 学员跑了一周后，最常问的是"AI 还是不像我，怎么办？"——这部分回答这个问题。

**核心动作**：在 CLAUDE.md 上做三类升级。

**升级 1：补充"风格采样"到 03_我说过什么**（5 分钟）

**话术**：
> "AI 不像你，最大的原因是它没读过足够多『你的原话』。一个 5 段的 CLAUDE.md 不够，AI 需要看你 30+ 段真实的写作。"

**操作**：
```
1. 在 wiki/03-content-lib/风格采样/ 新建文件
2. 粘贴 5-10 段你的代表作原文（公众号/朋友圈都行）
3. 在 CLAUDE.md 第三节加一句："风格请深度参照 wiki/03-content-lib/风格采样/"
```

---

**升级 2：补充"边界例子"到 CLAUDE.md**（5 分钟）

**话术**：
> "AI 不知道哪些话不能说，是因为你只列了禁忌词，没给反例。"

**示范**：
```markdown
## 七、边界规则（升级版）
- 涉及对外发布的内容必须我确认
  - 反例：龙虾自己把朋友圈发出去 → 永远先问我
- 不编造数据
  - 反例：龙虾说"我们 33 人实操营满意度 100%"——这个数据没出处 → 应该说"约 85% 完课率，5星好评"
- 不替我做承诺
  - 反例：龙虾说"3 天必出方案"——我没这个 SLA → 应该问我"这个时间承诺合适吗？"
```

---

**升级 3：增加"主动提问触发器"**（5 分钟）

**话术**：
> "好的 AI 助手不是只会回答，而是会**主动问你**。"

**示范**（加到 CLAUDE.md）：
```markdown
## 十一、主动提问触发器
当遇到以下情况，AI 主动停下来问我：
1. 用户原话引用 → "这段话是 [谁] 说的？我可以引用吗？"
2. 涉及具体金额/数字 → "这个数据来源是？"
3. 涉及他人姓名 → "这个名字脱敏吗？"
4. 写得太好但不像我 → "这段是不是 AI 味太重？要不要重写？"
```

## Part 4｜配置 AI 日报 + 每日工作汇总（15 分钟）

> **教学策略**：这是直播 2 学员"获得感最高"的部分——直接看到自动化在跑。

### ☀️ 4.1 配置每日 AI 日报（8 分钟）

**话术**：
> "明天早上 9 点，你的微信会自动收到一条 AI 日报。**不是订阅，不是推送，是你自己的 AI 助理给你写的。**"

**演示步骤**：

```yaml
打开 WorkBuddy → Automation → 新建任务

任务名：每日 AI 日报
触发：每天 09:00（北京时间）
执行：
  1. 调用 web_search 搜索：
     - "AI 最新进展 24小时内"
     - "AI 工具 新发布"
     - 关键词：大模型 / Agent / 知识库 / 内容创作
  2. 用 Claude 筛选 Top 3-5 条
  3. 每条关联到我的 M1/M2/M3 母题
  4. 写一段"这跟你有什么关系"
  5. 给一个具体行动建议
  6. 通过 ClawBot 发到我的微信
```

**输出格式**：
```
☀️ AI日报 | 2026-05-XX

1️⃣ [新闻标题]
📎 关联：M3-AI智囊团
💬 点评：解读这个新闻对你的意义...
🎯 你可以：一个具体行动建议

[......2、3 条同上......]

📊 一周总览：本周共 N 条相关新闻，3 条值得深读
```

**学员同步操作**：现场配置完成 → 测试触发一次（不等明天）。

## Part 5｜排错 Top 5 + Q&A + 作业（20 分钟）

### 🔧 5.1 直播 2 排错 Top 5（10 分钟）

> 提前在 PPT 备好截图+解决方案。

| # | 报错现象 | 原因 | 解决 |
|---|---|---|---|
| **1** | 微信发消息给龙虾**无响应** | ClawBot 服务掉线 / WebSocket 断了 | ① 检查 WorkBuddy 是否运行 ② 重启 ClawBot 服务 ③ 等 1-2 分钟自动重连 |
| **2** | 龙虾回复"**找不到相关资料**" | wiki/index.md 还没建好 / raw 为空 | ① 确认 wiki/index.md 存在且有内容 ② 确认 raw 下至少有 1 个文件 ③ 手动让 AI 跑一次 Ingest |
| **3** | Automation 定时任务**没触发** | 时区错 / 任务被禁用 | ① 检查时区设为 Asia/Shanghai ② 检查任务状态是否 Active ③ 手动触发一次测试 |
| **4** | LLM 回复**完全不像你** | CLAUDE.md 风格描述不够 / 没风格采样 | ① 在 03_我说过什么/风格采样/ 加 5-10 段你的原文 ② 在 CLAUDE.md 第三节加禁忌词 |
