## Slide 5｜分工对比表

| 工具             | 干什么                | 不干什么            |
| -------------- | ------------------ | --------------- |
| **Obsidian**   | 看/编辑/搜索本地 .md 文件   | 不发微信、不调AI、不跑工作流 |
| **WorkBuddy**  | 跑工作流、调度Skills、读写文件、调AI、处理消息 | ClawBot只是它的一个入口 |
| **微信 ClawBot** | 微信里收发消息的入口 → 把消息转给WorkBuddy处理 | 不自己做AI计算，只做桥梁 |

## Slide 6｜为什么分三层？—— 工程上的「关注点分离」

**类比**：

> 「你公司的 IT 部门会让你的电脑上同时运行『打印机+邮件+文件柜+ChatGPT』在同一个软件里吗？不会。
> **每个工具做自己最擅长的事，组合起来才稳定。**」

**打通前 vs 打通后**：

```
打通前（直播1之后你的状态）：
  微信(碎片) ←→ Obsidian(孤岛) ←→ AI(陌生人)
  三者各干各的，你手动搬运，累且容易断

打通后（这节课之后）：
  微信 ←→ ClawBot(信使) ←→ WorkBuddy(助理) ←→ Obsidian(底座)
  你只负责「发微信」和「确认」，中间全自动
```

     关联到「AI不懂我」这个高频痛点」
```

**金句**：
> 「你只发了一条微信，AI干了5件事。这就是『AI帮你工作』的具象。」

## Slide 8｜核心区别：其他AI帮你「想」，ClawBot帮你「干」

**这是最重要的认知升级：**

```
┌─────────────────────────────────────────────────────────┐
│                                                           │
│   🤖 大部分AI工具（ChatGPT/Claude/豆包/DeepSeek）         │
│                                                           │
│   帮你「想」                                              │
│   ─ 回答问题                                              │
│   ─ 写文章初稿                                            │
│   ─ 给建议、出方案                                        │
│                                                           │
│   但：说完就完了，不会帮你执行                           │
│                                                           │
├─────────────────────────────────────────────────────────┤
│                                                           │
│   🦞 ClawBot + WorkBuddy（基于OpenClaw底座）            │
│                                                           │
│   帮你「干」                                              │
│   ─ 收到消息 → 自动入库到Obsidian                      │
│   ─ 识别意图 → 调用Skills执行工作流                     │
│   ─ 定时任务 → 自动推送日报/汇总                        │
│   ─ 写完文件 → 帮你更新到对应房间                       │
│                                                           │
│   干完才告诉你结果                                       │
│                                                           │
└─────────────────────────────────────────────────────────┘
```

**类比**：

```
其他AI = 你的「军师」
  └─ 诸葛亮：帮你出主意、分析问题、想方案
       说完你就知道了，然后你自己去做

ClawBot = 你的「数字员工」
  └─ 直接帮你执行：入库、更新、推送、调度
       干完才告诉你结果
```

**为什么ClawBot能「干活」？**

```
因为它背后有 WorkBuddy 的 Skills 工作流引擎：
  你发消息 → OpenClaw底座识别意图 → WorkBuddy调度Skills →
  Skills读写Obsidian文件 → 返回结果给你

OpenClaw = 帮你接通微信的「管道」
WorkBuddy = 帮你执行任务的「引擎」
ClawBot = 在微信里帮你跑腿的「员工」
```

**金句**：
> 「别人用AI是『想完自己干』，
> 你用ClawBot是『说一声它就干了』。」

Claude Code（补充工具）→ 在桌面帮你「直接跑脚本/改文件」
      ↓ 和WorkBuddy互补
```

## Slide 9｜WorkBuddy 和 Claude Code 的关系 —— 不是一个，是两个

**类比**：

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│   🐾 WorkBuddy                                     │
│   像：一个 24小时值班的主管                          │
│   - 定时任务自动跑（每天9点发日报）                  │
│   - 微信消息自动响应                               │
│   - 调度多个Skills工作流                            │
│   - 你不在线，它也在后台跑                          │
│                                                     │
├─────────────────────────────────────────────────────┤
│                                                     │
│   💻 Claude Code                                   │
│   像：一个随时听你话的桌面秘书                       │
│   - 你在终端/桌面直接说「帮我整理文件夹」            │
│   - 直接读写本地文件、改代码                        │
│   - 每次主动启动，你说什么它做什么                  │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**怎么选？**

```
想做什么？
    │
    ├─ 在桌面随手让AI帮我整理文件、改文档、跑脚本
    │   → Claude Code（即时、本地、每次主动启动）
    │
    ├─ 我发条微信给AI，它自动帮我入库/回复
    │   → WorkBuddy + ClawBot（24小时在线、自动触发）
    │
    └─ 定时任务（每天9点AI日报、每周一汇总）
        → WorkBuddy的Automation模块
```

**一句话**：
> 「Claude Code 是你的桌面贴身秘书，WorkBuddy 是你的24小时数字员工。两个都用，互为补充。」
