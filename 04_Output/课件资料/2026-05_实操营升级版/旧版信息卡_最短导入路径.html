二、最短导入路径（4 步走）

Step 1：把你的旧版信息卡导出（5 分钟）

不管你之前用的是飞书 / Notion / 印象笔记 / Obsidian，都导出为单一文件： | 你之前用什么 | 怎么导出 | |---|---| | 飞书 | 选中所有信息卡 → "导出 Word" 或 "导出 Markdown" | | Notion | 右键页面 → Export → Markdown & CSV | | 印象笔记 | 选中笔记 → 导出为 HTML / PDF | | Obsidian | 直接拷贝 .md 文件 | | Word/PDF | 直接用现有文件 | 输出文件：1 个文件夹，里面有 11 个独立文件（或 1 个汇总文档）

Step 2：放到你的 ob-大脑 raw/ 目录（1 分钟）

`bash

在你的 ob-大脑 根目录下

mkdir -p raw/旧版信息卡

把刚才导出的所有文件拷进去

cp -r [导出的文件夹]/* raw/旧版信息卡/ `

Step 3：启动 Claude Code 跑导入指令（10-20 分钟）

`bash cd "[你的 ob-大脑 路径]" claude ` 启动后，完整复制下面这段指令粘给 Claude Code： ` 我有一份 3-4 月实操营填好的旧版信息卡（11 张），已经放到 raw/旧版信息卡/。 请帮我按 v2 路径 C 分工法则重新整理到 wiki/： 【迁移规则】 1. 旧版【模板1 一句话定位卡】 → wiki/01_我是谁/信息卡/2026-05-XX_定位卡.md 2. 旧版【模板2 故事线卡】 → wiki/01_我是谁/信息卡/2026-05-XX_故事线卡.md 3. 旧版【模板3 价值观与边界卡】 → wiki/06_我怎么想/信息卡/2026-05-XX_价值观边界卡.md 4. 旧版【模板4 给 AI 的使用说明书】 → 提炼内容追加到根目录 CLAUDE.md（不再单独成卡） 5. 旧版【模板5 方法论卡】（可能多张）→ wiki/02_我会什么/信息卡/ 6. 旧版【模板6 案例卡】（可能多张）→ wiki/02_我会什么/信息卡/ 7. 旧版【模板7 内容资产卡】 → wiki/03_我说过什么/2026-05-XX_内容资产卡.md 8. 旧版【模板8 主题包卡】（可能多张）→ wiki/03_我说过什么/主题包/ 9. 旧版【模板9 用户画像卡】（可能多张）→ wiki/05_我面对谁/信息卡/ 10. 旧版【模板10 用户问题库卡】 → wiki/05_我面对谁/信息卡/ 11. 旧版【模板11 决策原则卡】（可能多张）→ wiki/06_我怎么想/信息卡/ 12. 旧版【模板13 产品卡】（可能多张）→ wiki/04_我卖什么/信息卡/ 【格式调整】 - 删掉每张卡里的"给 AI 的提醒"段（调用场景/调用规则/禁止事项/语气要求） → 这些统一提炼后追加到 CLAUDE.md - 保留卡片主体内容（对象/场景/问题/结论 等） - 每张卡顶部加 frontmatter（type / source / migrated_from / date） - 文件名格式：YYYY-MM-DD_[卡片类型]_[简短主题].md 【同步动作】 - 每张卡迁移完，追加索引行到 wiki/index.md - 在 wiki/log.md 记一笔：[Migrate] 旧版模板X → wiki/Y/Z.md 【完成后告诉我】 - 总共迁移了几张卡 - 哪些素材没找到合适的房间（待我手动确认） - CLAUDE.md 的"调用规则"部分新增了什么 `

Step 4：你审核 + 微调（10 分钟）

跑完后，按这个清单检查： ` □ 5 间房（01,02,03,05,06）每间都有 ≥1 张卡 □ wiki/index.md 索引行齐全 □ wiki/log.md 有迁移记录 □ CLAUDE.md 末尾追加了"调用规则"部分 □ 所有卡的"对象 / 场景 / 问题"等核心字段没丢 □ 跑一次 Query 验证："用我的 wiki 给我写一段自我介绍" ` 通过 = ✅ 迁移完成。

四、迁移后会改变什么？

✅ 你会得到的好处

1. wiki 结构跟全班统一——以后蓉蓉子发的 prompt / skill 你都能直接用 2. CLAUDE.md 更精简——所有"调用规则"集中在一个地方，不分散 3. AI 调用更快——不用每次读 11 张卡的"给 AI 的提醒"，直接读 CLAUDE.md 4. 跟你的小龙虾🦞 兼容——直播 2 的龙虾默认按 v2 结构调用

⚠️ 你需要注意的

1. 旧版的"给 AI 的提醒"段被合并了——别担心，内容都在 CLAUDE.md，没丢 2. 如果你旧版填得很简略（比如只填了 5-6 张）——AI 提炼后房间会稀疏，建议跑 Day 4-5 喂公众号文章补 03 / 05 房间 3. 故事线卡可能需要小更新——3-4 个月过去了，你可能有新的转折点

五、常见问题

Q1：我之前填的卡是中英混合 / 排版乱，AI 能识别吗？

A：能。Claude 对中英混合 + 各种 Markdown 排版兼容性很好。如果有特殊格式（如表格嵌套），跑完后查一下，必要时让 AI 重排。

Q2：旧版我有些卡填得很差，要不要重填？

A：不要！ 让 AI 先按"差版本"导入，跑通后你就知道哪些卡需要补。v1 跑通 > v3 完美——这是这次实操营反复强调的。

Q3：我旧版根本没填满 11 张（比如只有 5 张），还能迁移吗？

A：可以。AI 会按"有什么迁什么"的原则处理。剩下没填的房间，跟着 5 月实操营节奏喂料让 AI 补。

Q4：迁移过程中 AI 改错了我的内容怎么办？

A：3 道防线： - raw/旧版信息卡/ 是原始档案，永远不动 - log.md 有迁移记录，可以追溯 - 如果错了，跟 AI 说"wiki/X.md 第 Y 段不对，请改成原始版本"

Q5：我能不能不迁移，继续用旧版？

A：可以，但不推荐。原因： - 直播 2/3 的 prompt 都按 v2 结构写的，你用旧版要自己翻译 - 龙虾的 SKILL.md 默认调 v2 路径 - 长期看，跟全班结构不一致会越来越累
