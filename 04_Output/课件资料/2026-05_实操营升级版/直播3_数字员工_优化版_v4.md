## P06 · Skill 到底是什么？和数字员工有什么关系？（10min）

### 一个最常见的误解

> "很多人以为 Skill = 数字员工。不对。
> **Skill 是零件，数字员工是会用零件的工人。**"

| 对比维度 | Skill（零件） | 数字员工（工人） |
|---|---|---|
| **是什么** | 一个 .md 说明书 | 一个能自主决策的角色 |
| **能干什么** | 完成一个具体任务 | 判断该调用哪个Skill、按什么顺序 |
| **举例** | `topic-generator` = 切菜刀 | 🐾 石榴 = 厨师，知道先切菜再炒菜 |
| **配置方式** | 写 SKILL.md + prompts/ | 写 CLAUDE.md + 调用多个Skill |
| **关系** | 数字员工调用Skill | → |

### Skill 到底长什么样？

> "本质就是——**你给 AI 一份说明书**。一个 .md 文件。"

```
my-skill/
├── SKILL.md          ← 触发词 + 说明
└── prompts/
    └── main.md       ← 核心提示词（30行以内）
```

**示例**：
```markdown
# 行业周报生成器
## 触发词
"帮我出本周行业周报" / "周报"
## 说明
基于我的 Wiki + 本周素材自动生成行业周报
```

### 💬 金句

> "**Skill 不是写代码，是写说明书。会写文档你就会写 Skill。一个 Skill 不到 30 行。**"

> 配套参考文件：《Skill创建指南》+《5类数字员工模板示意》（学员模板文件夹内）

## P07 · 5步工作流总览（3min）

### 核心链路

```
         微信发碎片
              ↓
         ① 出选题
       （基于Wiki母题+本周素材）
              ↓
         ② 出大纲
       （GAP框架：问题→拆解→原则→行动）
              ↓
         ③ 出初稿
       （调03风格+01身份+02方法论）
              ↓
         ④ 你改20%
       （语气/故事/金句 —— AI改不了的）
              ↓
         ⑤ 一键分发
       （公众号草稿 + 小红书图文）
              ↓
         好内容反哺 Wiki
```

### 关键原则

- **用真实素材跑演示**：当下真实的碎片，不用demo假数据
- **不快进不剪辑**：完整跑一遍，让学员看到真实耗时（~30分钟）
- **AI写80%，你改20%**：这20%就是"为什么是你"的部分

### Step 2-3 → 选定主题，出大纲+初稿（15min）

**操作**：选定1个选题 → GAP框架出大纲 → 调风格出初稿

**GAP框架**：
- **Gap（问题）**：痛点描述
- **Analysis（拆解）**：误区分析
- **Principle（原则）**：方法论
- **Action（行动）**：具体步骤

**演示**：
```
选定：1️⃣ 【M3】"知识库和我有什么关系"

↓ AI 出大纲 ↓

# 知识库和我有什么关系？

## Gap（问题）
- 搭了知识库，但不知道对我有什么用
- 感觉是给老师交作业

## Analysis（拆解）
- 误区1：知识库=笔记仓库 → 不是，是AI的说明书
- 误区2：知识库是给人看的 → 是给AI读的
- 真相：知识库是你的"数字大脑"，让AI替你工作

## Principle（原则）
1. 知识库的第一读者是AI，不是你
2. 结构化 > 数量多
3. 用一次就值回票价

## Action（行动）
1. 今天用5步法出一篇"我和我的数字员工"
2. 发布到1个平台
3. 感受一下从碎片到稿件只要30分钟

↓ AI 出初稿（调用03风格+01身份）↓

[1500字初稿，语气仿蓉蓉子]
```

**💬 金句**："你平时写一篇文章要3小时。现在，大纲+初稿一共15分钟。前提是你的底座搭好了。没底座，AI写出来的就是'正确的废话'。"

---

### Step 4-5 → 你改20%，一键分发（15min）

**四维润色检查表**（只改这4个地方）：

| 检查项 | 看什么 | 怎么改 |
|---|---|---|
| ☐ 语气像我吗？ | 读出来听听 | 加1-2句你自己的口头禅 |
| ☐ 专业准确吗？ | 有没有外行话 | 改成你的表达方式 |
| ☐ 用户视角吗？ | 能不能戳中读者 | 加1个真实学员故事 |
| ☐ 价值观一致吗？ | 有没有越线 | 删掉"正确的废话" |

**💬 金句**："注意——不是从头改到尾。只改这4个维度。**AI写80%，你改20%。这20%才是'为什么是你'。**"

**一键分发**：
- **公众号草稿**：保留原文风格，完整正文 → 发到草稿箱
- **小红书图文**：emoji密、节奏快、第一句钩子、带标签
- **好内容反哺Wiki**：新案例入库、风格采样更新、选题库新增

> "不同的输入走不同的路——这就是路由。内容走生产流，灵感存入待处理，咨询记录更新痛点库。"
