# 当你每天都在追热点后，个人IP真正的主线：一份“母题清单”

很多个人 IP 之所以越写越累，不是因为你不够努力，而是因为你被一种“看似勤奋”的模式绑架了：每天追热点，输出节奏被平台牵着走，没有自己的主线。写得越多，越像在替平台打工——而不是在搭自己的事业。更隐蔽的是：你会开始怀疑自己是不是“不够有才华”。可真相往往更简单，也更刺耳：你不是不会写。你只是没有一条能把所有内容串起来的—— 母题主线 。  


01

热点不是问题， 没有主线才是问题  
写在前面：我不是反热点。热点本质上是一个“放大器”。它能把你原本就有的观点放大，也能把你本来就空的内核暴露得更空。真正致命的是：你把热点当成了 内容的发动机 。

  * 于是你会掉进一种循环：


  * 今天追一个热词


  * 明天换一个赛道


  * 后天再换一种人设

一个月过去，你写了 20 篇，但没有任何一篇能变成你的“代表作”。因为你缺的不是热点，缺的是一条能把热点“吞进去消化掉”的—— 母题 。  


02

什么是“母题”？它不是选题，是你的“长期三口井”  
在我自己的知识库内容系统里，除了内容资产库还增加了：母题清单/主题主线部分。写母题前，你要先回答这个问题：接下来 1–2 年，我最想反复讨论的 3 个问题是什么？这一步很关键，因为它会把“写什么”从无限可能，变成 有限但很深的三口井 。你会发现一个很神奇的变化：以前你写作像“抽奖”。今天状态好就写得出来，状态差就卡死。（我自己也经历过这个状态，输出靠灵感）有了母题以后，你写作更像“挖井”。你不是每天换地方开挖，而是在同一个地方不断往下走。深度，就是这样来的。信任，也是这样来的。  


03

母题清单怎么写？给你一个“人话版”模板  
别搞复杂。你只要写出 3–5 条就够了。我建议你用这个格式：我会反复讨论：  
  
因为我想帮（谁）解决（什么） 我愿意为这个问题花 3 年、5 年去迭代答案然后你可以从 3 个来源里“捞母题”：从你的经历里捞：你走过的弯路，就是你的主题矿问自己三个问题：

  1. 我最痛的一次崩溃/转折是什么？
  2. 我后来是怎么走出来的？
  3. 我现在最想把哪句“过来人提醒”递给别人？

（母题不是“我懂什么”，而是“我替别人受过什么罪”。）从你的用户里捞：他们反复问的，就是市场在敲门你不需要大数据。你只要回看：

  * 评论区高赞问题


  * 私信里重复出现的困惑


  * 咨询时对方最纠结的那句话

这些就是母题的原矿。从你的产品里捞：你最终要交付什么，就该长期写什么如果你未来要做课程/咨询/陪跑，你的母题最好能自然通向：

  * 方法论


  * 框架


  * 练习


  * 案例库

这样内容才会从“表达”变成“资产”。  


04

母题和热点怎么共存？答案是： 让热点做“场景”，母题做“主线”  
很多人误会了：有母题 = 不追热点。不是的。更高级的玩法是：母题是你的树干，热点只是长出来的叶子。在内容知识库里，我在目录里加的“长期想讲的 3–5 个母题”清单模版，核心就是用来确保内容不跑偏、有一致世界观。而选题真正轻松的秘密是这句：用「场景 + 主题」去检索今天写哪块。举个例子：

  * 热点：AI 又出新模型了


  * 你的母题：职业第二曲线 / 一人公司 / 可持续输出

那你就不需要写“这个模型多强”。你可以写：

  * “为什么大多数人学 AI 学到最后更焦虑？” （第二曲线）


  * “工具再多也救不了你：你缺的是可调用的内容资产” （输出系统）


  * “真正的护城河不是工具，而是你如何做选择” （决策风格）

同一个热点，别人写新闻，你写主线。这就是差距。  


05

把母题变成“可复用资产”的关键动作：入库  
我在另一篇里说过一句很重要的话：我们要搭的不是“好看的文件夹系统”，而是一套可以被反复调用、被 AI 放大的个人大脑结构。所以母题清单不是写完贴在备忘录里就结束了。你要把它放进你的知识库里，成为你内容系统的“导航栏”。然后从今天开始，养成一个动作：每写完一篇内容、做完一次咨询、结束一场项目，问自己一句：“这一块，应该放进我哪一间房？”你会慢慢从“发内容的人”，变成“搭底座的人”。  


06

给你一个今天就能做的小练习（10 分钟）  
拿出纸和笔，写下这三句：

  1. 我未来 1–2 年最想反复回答的问题是： （写 3 条）
  2. 我最想服务的人是： （越具体越好）
  3. 我想让对方发生的改变是： （从 A 到 B）

写完你就会发现：你根本不缺选题。你缺的是一条“不会跑偏的主线”。  


07

写在最后：你不是不够努力，你只是缺一份“母题清单”

母题清单，像买地基——  
你写的每一篇，都会回到你自己的土地上。
