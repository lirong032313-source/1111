# 实测：从微信里一键启动内容工厂，我打通了这套自动化流程

> 一个人 + 一个微信对话框 = 一个完整的内容生产系统

---

## 缘起：为什么要在微信里做内容？

做 IP 这两年，我有个很深的体会：

**灵感总是出现在最不方便的时候。**

- 地铁上刷到一条好内容，想写点什么，但电脑不在身边
- 睡前突然想到一个选题，怕忘了，只能发给自己微信
- 和朋友聊天聊到某个观点，觉得可以展开成文章，但转头就忙忘了

微信是我们每天打开次数最多的应用。如果能在微信里直接完成「从想法到文章」的闭环，那效率会提升多少？

抱着这个想法，我最近在 WorkBuddy 里搭建了一套「微信入口内容工厂」，并且打通了和 YouMind 内容引擎的联动。

今天分享下这套流程的实测体验。

---

## 我的内容工厂架构

先上架构图，方便理解：

```
微信对话框（入口）
    ↓
WorkBuddy（智能调度器）
    ↓
┌─────────────────────────────────────┐
│  意图识别：明确输入 / 模糊输入        │
│  路由分发：公众号 / 小红书 / 朋友圈   │
└─────────────────────────────────────┘
    ↓
知识库预检索（飞书/个人素材）
    ↓
内容生成（本地 LLM / YouMind 引擎）
    ↓
结果输出 + 自动沉淀
```

简单说：**我在微信里发一句话，系统就能自动判断我要做什么，然后完成内容生产。**

---

## 实测场景一：明确输入，直接出稿

**我的输入：**
> "写一篇关于个人知识库好处的公众号文章"

**系统处理流程：**

1. **意图识别** → 判定为"明确输入"（主题+平台都清晰）
2. **知识库检索** → 自动读取我的 IP 人设、内容主线、历史素材
3. **内容生成** → 按公众号格式输出完整文章
4. **结果沉淀** → 自动保存到工作目录，方便后续编辑

**输出效果：**
- 标题、摘要、正文结构完整
- 语气符合我的 IP 调性（亲切、真实、有洞察）
- 内容融入了我的核心观点（M3 AI知识库主线）

**耗时：**从发送到拿到成稿，不到 30 秒。

---

## 实测场景二：模糊输入，智能选题

**我的输入：**
> "最近有什么选题灵感？"

**系统处理流程：**

1. **意图识别** → 判定为"模糊输入"，需要选题建议
2. **知识库分析** → 读取我的三大内容主线（M1/M2/M3）
3. **选题生成** → 基于我的定位，生成 3 个可选选题

**输出示例：**
- 选题 1：《30岁转型自由职业，我踩过的 5 个坑》（M1 成长日记）
- 选题 2：《如何把一次咨询变成可复用的产品》（M2 小而美产品）
- 选题 3：《我的 AI 知识库搭建实录：从 0 到 1 的全过程》（M3 AI知识库）

每个选题都标注了适合的平台、预计风格、承接产品。

**体验感受：**比我自己想选题还准，因为它基于我的历史内容和定位来推荐。

## 打通 YouMind：内容引擎升级

WorkBuddy 本地生成已经很好用了，但我还想更进一步——**让专业内容引擎来加持**。

于是我接入了 YouMind 内容工厂：

```
微信消息
    ↓
WorkBuddy 路由层
    ↓
YouMind 内容引擎（API 调用）
    ↓
专业级内容生成 + 自动沉淀到 Board
```

**接入后的变化：**

| 维度 | 本地生成 | YouMind 引擎 |
|------|---------|-------------|
| 内容质量 | 通用水平 | 针对平台专门优化 |
| 知识库联动 | 飞书文档 | YouMind Board（更智能的上下文） |
| 沉淀方式 | 本地文件 | 自动归档 + 标签分类 |
| 越用越懂你 | 有限 | 基于 Board 历史持续优化 |

**实测对比：**

同一主题「WorkBuddy 体验」，分别用两种方式生成：

- **本地生成**：结构完整，观点清晰，符合预期
- **YouMind 生成**：更懂公众号的"爆款逻辑"，标题更有吸引力，案例更贴近读者痛点

两者各有优势，我现在会根据内容重要性选择调用方式。

---

## 这套流程给我的改变

**1. 内容生产的门槛降低了**

以前写一篇公众号文章，需要：
- 打开电脑
- 打开文档
- 构思结构
- 开始写作
- 反复修改

现在：微信里发一句话，30 秒后拿到初稿，我只需要做「精修」和「加料」。

**2. 灵感不再流失**

任何时间、任何地点，有想法就发给微信，系统会自动处理。我只需要在方便的时候审核和发布。

**3. 内容一致性提升了**

因为系统记住了我的 IP 人设、内容主线、表达风格，每次生成的内容都"像是我写的"，而不是"AI 写的"。

---

## 如果你也想搭建这套系统

核心需要三部分：

**1. 入口层** — 微信机器人（WorkBuddy/OpenClaw）
- 接收消息，调用后端技能

**2. 调度层** — 意图识别 + 路由分发
- 判断用户要什么，分发到对应处理流程

**3. 引擎层** — 内容生成（本地 LLM / YouMind API）
- 基于知识库上下文，生成符合人设的内容

**4. 沉淀层** — 自动归档到知识库
- 每次输出都沉淀下来，让系统越用越懂你
