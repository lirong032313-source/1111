
## 🗂️ 目录导航

### 📥 00_Inbox / 收件箱 — **临时候转，定期清空**

| 说明 | 规则 |
|---|---|
| **用途** | 所有新进来的东西先扔这里，不定性不清不往 Raw 放 |
| **清空节奏** | 每周/每两周处理一次，处理完移走或删除 |
| **什么放这里** | 随手存的链接、截图、临时笔记、未分类文件 |
| **什么不放这里** | 已确认的原始素材（直接进 Raw） |

**规则**：
- ⚠️ **不要让 Inbox 变成垃圾场** — 定期清空是铁律
- 处理完的东西：有价值→归类到对应层级，没价值→直接删
- AI 启动扫描时**先看 Inbox 有没有新东西**

### 📦 01_Raw / 原料层 — **只读不改**

| 子目录        | 存什么           | 示例       |
| ---------- | ------------- | -------- |
| `咨询逐字稿/`   | 客户1v1咨询录音转文字  | TXT/Word |
| `课程逐字稿/`   | 直播/训练营课堂实录    |          |
|            |               |          |
| `学员文料/`    | 学员提交的作业/反馈    | Word/PDF |
| `已处理清单.md` | ✅ 已处理的Raw文件索引 | **启动必读** |

**规则**：
- ⚠️ AI **只能读取**这层，**绝不修改**
- 所有原始素材永久保存，不删不改
- 不确定的信息标记 ⚠️ 待确认，禁止补编

## 🏷️ 文件命名规范

```
YYYY-MM-DD_类型_主题.md
```

**示例**：
- `2026-04-26_原始素材_个人故事-我是谁.md`
- `2026-04-26_咨询逐字稿_AI CEO能量团队.md`
- `2026-04-26_公众号文章_个人知识库的好处.md`
- `2026-04-26_方法论文档_小而美产品设计SOP.md`

**类型词表**：

| 层级 | 可用类型 |
|---|---|
| Raw | 原始素材、咨询逐字稿、课程讲义、灵感碎片、学员案例、行业文章 |
| Wiki | 知识库、方法论、客户画像、金句库、内容选题、产品体系 |
| Output | 公众号文章、客户方案、课程脚本、小红书笔记、招募文案 |

# 🚀 启动扫描工作流（每次操作前执行）

1. **读本文件**（必读）— 了解完整架构
2. **检查 `00_Inbox/` 有无新文件** — 有则先分类处理
3. **读 `01_Raw/已处理清单.md`**（对比用）— 跳过已处理
4. **只处理不在清单里的新文件**— 处理完追加到清单
