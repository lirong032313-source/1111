## 四、🎯 整体工作分工法则（全程都用）⭐
> **这是 21 天最重要的一条原则。** 看完这一节，你才明白这次实操营和别的 AI 课的根本区别。
### 谁做什么？
```plaintext
┌──────────────────────────────────────────────┐
│   ✋ 你必须手写（约 3 张地基卡，AI 替代不了）   │
├──────────────────────────────────────────────┤
│  ❶ 01_我是谁/一句话定位.md                    │
│     → 必须自己想清楚"我是谁"                   │
│  ❷ 06_我怎么想/核心价值观.md                  │
│     → 必须从你内心来                          │
│  ❸ CLAUDE.md 第三节"写作风格"                 │
│     → 你和 AI 的契约，必须你写                │
└──────────────────────────────────────────────┘

┌──────────────────────────────────────────────┐
│   🤖 AI 提炼（约 80% 的卡片）                  │
├──────────────────────────────────────────────┤
│  你只管：把素材丢进 raw/                      │
│  AI 自动：提炼 → 写信息卡 → 更新 index.md     │
│                                              │
│  典型场景：                                    │
│  • 02_我会什么 ← 你的方法论文章 / 课件         │
│  • 03_我说过什么 ← 公众号 / 朋友圈历史         │
│  • 04_我卖什么 ← 产品说明书 PDF               │
│  • 05_我面对谁 ← 咨询记录 / 学员反馈           │
│  • 案例库 ← 已发爆款反向沉淀                  │
└──────────────────────────────────────────────┘

```

### 核心金句（请背下来）
> "**地基卡你必须手写——因为这是 AI 替代不了的'你成为你'的过程。****装修部分让 AI 干——因为这是 AI 比你快 100 倍的事。**"
### 🚨 反例（绝对不要这样做）
❌ **让 AI 替你写"一句话定位"**→ AI 写的会非常空洞，因为没有你的真实判断
❌ **手写所有 80 张信息卡**→ 会累死，21 天填不完，违背 LLM Wiki 的核心范式

#### Day 2-3：丰富 02 我会什么
**任务**：把你的方法论沉淀给 AI。
**操作**：
1. 找出你过去写过的"方法论型"文章 / 课件 PDF（3-5 篇即可）
1. 丢到 `raw/articles/` 文件夹
1. 在 Claude Code /workbuddy里说：请帮我提炼wiki 我会什么方法论信息卡
1. AI 提炼完后，你**审核**生成的信息卡——不像你的地方告诉 AI 改
**验收**：02 房间至少 2 张方法论信息卡。

#### Day 4-5：丰富 03 + 05
**任务**：让 AI 学会你的语气 + 看清你的用户。
**Day 4 操作**（03 房间）：
1. 把公众号历史文章导出（用obsidian的网页裁剪）
1. 让 AI 提炼：
  - 30 段最有"你味"的段落 → 03/风格采样/
  - 20 句标志性金句 → 03/金句库/
  - M1/M2/M3 各 5-10 篇代表作 → 03/代表作清单.md
**Day 5 操作**（05 房间）：
1. 找你过去的咨询记录 / 学员反馈 / 用户原话
1. 丢到 `raw/chats/` 或 `raw/notes/`
1. 让 AI 提炼：用户原话 + 痛点分类 → 05/信息卡/
**验收**：03 风格采样 ≥30 段；05 痛点卡 ≥1 张。
---

#### Day 6：手写 06 核心价值观
**任务**：这是地基卡 #2 ——AI 替你写不了。
**晚上找一个安静时间，手写**：
```markdown
# 06_我怎么想/核心价值观.md

## 我做事的 3 条原则
1. ____
2. ____
3. ____

## 我不做什么（边界）
- ____
- ____

## 我跟主流不一样的地方
- 别人都说 ___，但我认为 ___

```

写完后让 AI 跑一次 Query 验证：
```plaintext
基于我所有 wiki，给我写一段 200 字的自我介绍。

```

如果 AI 写出来"像你" = ✅ 你的价值观和风格已经被 AI 吸收。

**里程碑动��**：

<text color="purple" underline="true">**附升级版的CLAUDE.md 工作流规则（细节根据自己的实际情况调整下）：**</text>
```plaintext
十、工作流规则

Ingest（摄入）— 新素材入库
当你帮我摄入新素材时：
识别类型：判断属于哪个房间（01-06）
提取要点：2-5 条核心摘要，写入「信息卡/」
详细内容：完整内容存入「详细内容/」
更新索引：同步更新对应房间的 index.md
关联母题：判断属于 M1/M2/M3 中的哪个方向

Query（查询）— 向知识库提问
当我向知识库提问时：
定位相关页面：先读 index.md 找到相关房间
读取信息卡：优先引用「信息卡/」的内容
综合回答：引用时标注来源，如 [来源：01_我是谁/信息卡/个人定位.md]
诚实边界：不确定的明确说，不编造

Lint（体检）— 定期检查
⚠️ 矛盾信息：同一问题不同页面说法不一致
🗂️ 孤立页面：没有被任何地方链接的页面
❌ 缺失概念：被多次提到但没有独立页面的主题
```

## 🔟 给你的 3 个开营前心理建设
### 1️⃣ 不要追求"一次写完美"
v1 永远不完美。**跑通 > 完美**。21 天里你会迭代很多版。
### 2️⃣ 不要替 AI 操心
AI 写错的地方就指出来让它改，不要自己默默重写。**这是路径 C 的核心——不是你帮 AI，是你监督 AI 帮你**。
### 3️⃣ 不要孤军奋战
跟搭子说"我卡住了"不丢人。**21 天里，能不能成不取决于你多努力，取决于你被卡住时有没有人接住你**。
