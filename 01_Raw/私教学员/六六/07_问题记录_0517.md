# 我的 LLM Wiki 规则书

> 最后更新：YYYY-MM-DD
> AI 阅读优先级：每次对话开始时必读

## 一、我是谁（一句话定位）
- 名字：
- 一句话：[职业身份] + [服务对象] + [独特价值]
  例：我是 [xx教练]，帮 [30+职场人] [打造小而美产品]
- MBTI/性格关键词：

## 二、六间房含义（AI 用这个分类素材）
| 房间 | 装什么 | 不装什么 |
|---|---|---|
| 01_我是谁 | 身份/故事/价值观 | 工作方法 |
| 02_我会什么 | 方法论/案例/能力 | 个人感悟 |
| 03_我说过什么 | 文章/金句/原话 | 别人说的 |
| 04_我卖什么 | 产品/价格/服务 | 想做未做的 |
| 05_我面对谁 | 用户画像/痛点/原话 | 我自己的话 |
| 06_我怎么想 | 框架/原则/边界 | 具体执行 |

## 三、写作风格（AI 模仿这个语气）
- 整体感觉：（如：温暖专业 / 幽默轻松）
- 标志性表达：（2-3 个例子）
- 禁忌词：（绝对不出现）

## 四、母题清单（AI 写内容时按这个分）
- M1：（你的故事方向）
- M2：（你的产品方向）
- M3：（你的方法论方向）


## 五、Ingest 规则（AI 整理素材的方法）
1. 收到素材 → 判断属于哪个房间
2. 提取 2-5 条要点 → 写信息卡到 wiki/对应房间/
3. 详细原文存到 raw/ 对应类型/
4. 同步更新 wiki/index.md（追加一行）
5. 在 wiki/log.md 记录操作

## 六、Query 规则（AI 回答问题的方法）
1. 先读 wiki/index.md 找相关房间
2. 优先引用"信息卡"
3. 回答时标注来源 [来源：房间/文件名]
4. 不确定的明说"我不确定"，不编造

## 七、边界规则（AI 必须守的红线）
- 涉及对外发布的内容，必须我确认
- 不泄露学员/咨询客户的隐私
- 不编造我没说过的话或数据
- 风格不像我时主动停下问我

Lint（体检）— 定期检查
⚠️ 矛盾信息：同一问题不同页面说法不一致
🗂️ 孤立页面：没有被任何地方链接的页面
❌ 缺失概念：被多次提到但没有独立页面的主题
