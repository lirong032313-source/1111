# 标签体系 Schema

> 六间房语义网络的核心：让信息卡之间形成网状连接
> **最后更新**：2026-05-18

---

## 为什么需要标签体系？

| 之前 | 现在 |
|------|------|
| 信息卡只属于某个房间 | 信息卡可以同时属于多个维度 |
| 查询只能找到「一张卡」 | 查询可以找到「一张网」 |
| 六间房是六个独立仓库 | 六间房是一张互相连接的语义网 |

## 标签分类（四类）

### 🏠 房间标签（必须）

每个信息卡必须标注归属房间：

```
#1号房-我是谁
#2号房-我会什么
#3号房-我说过什么
#4号房-我卖什么
#5号房-我面对谁
#6号房-我怎么想
```

### 📚 母题标签（推荐）

蓉蓉子的三大内容母题：

```
#M1-第二曲线成长      ← 个人成长、转型故事、觉醒故事
#M2-小而美产品        ← 产品化思路、变现路径、商业模式
#M3-AI智囊团          ← 知识库搭建、AI工具、效率提升
```

### 🎯 能力标签（推荐）

蓉蓉子的核心专业能力：

```
#知识库搭建      ← 六间房 / LLM Wiki / Obsidian
#个人品牌        ← IP定位 / 人设打造 / 内容矩阵
#产品设计        ← 小而美产品 / 定价策略 / 交付SOP
#AI赋能          ← AI工具使用 / 提示词 / 工作流
#内容创作        ← 写作框架 / 金句 / 风格采样
#教练陪伴        ← ICF教练技术 / 私教陪跑
#企业解决方案    ← 10年+企业售前经验
```

### 👥 人群标签（可选）

```
#IP伙伴           ← 想打造个人IP的创作者
#自由职业者       ← 正在探索自由职业的人
#职场转型者       ← 想从职场转型到自由职业
#内容创作者       ← 需要内容生产方法的人
#知识工作者       ← 想搭建知识体系的人
```

### 📦 产品标签（可选）

```
#IP六间房模板     ← ¥199 入门工具
#21天实操营       ← ¥1899 21天陪跑
#1v1咨询         ← ¥999 深度咨询
#3个月私教        ← ¥16800 深度陪跑
```

### 📥 来源标签（可选）

```
#直播内容         ← 来自直播逐字稿
#咨询案例         ← 来自1v1咨询
#私教素材         ← 来自私教逐字稿
#共创会素材       ← 来自共创会
#原创提炼         ← 蓉蓉子原创方法论
#外部素材         ← 外部学习资料
#灵感碎片         ← 微信聊天 / 突然的想法
```

## 信息卡 frontmatter 模板

```yaml
---
title: [信息卡标题]
room: [房间归属]

tags:
  rooms:    [#房间标签]
  motifs:   [#母题标签]       # 可选
  skills:   [#能力标签]       # 可选
  audience: [#人群标签]       # 可选
  products: [#产品标签]       # 可选
  source:   [#来源标签]       # 可选

ai_trigger: [AI调用场景]      # 如：搭知识库 / 写文章 / 做咨询

related:                          # 同房间关联（Obsidian wiki链接）
  - "[[另一张信息卡]]"

created: YYYY-MM-DD
---
```

## 标签组合示例

### 场景：写一篇「知识库搭建」的文章

AI 看到 `#知识库搭建` 标签，可以横向拉取：

```
#知识库搭建
├── 2号房 → 六间房框架、LLM Wiki三层架构
├── 3号房 → 相关金句（"结构比内容更重要"）
├── 5号房 → 用户痛点（"知识散落没有体系"）
├── 6号房 → 决策原则（"做减法"）
└── 产品 → IP六间房模板、21天实操营
```

### 场景：写一篇「个人品牌」的文章

AI 看到 `#个人品牌` 标签，可以横向拉取：

```
#个人品牌
├── 1号房 → 蓉蓉子的个人品牌故事
├── 2号房 → 个人品牌方法论
├── 3号房 → 相关金句、风格采样
├── 5号房 → 用户画像（想打造IP的30-35岁人群）
└── 产品 → 21天实操营、1v1咨询
```

## 使用规则

1. **房间标签必须**：每张卡至少有一个房间标签
2. **标签不要过多**：每个维度最多 2-3 个，过多等于没有重点
3. **定期维护**：每月检查标签使用情况，合并/清理
4. **AI 查询时**：说「找所有 #XX 标签的内容」，AI 自动横向拉取
