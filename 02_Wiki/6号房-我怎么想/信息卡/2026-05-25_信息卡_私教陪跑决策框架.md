## 三阶段判断法

问三个问题决定当前阶段：

1. **底座有没有？** → 没有 → 第一阶段：搭知识库 + 上线第一个智能体
2. **流程顺不顺？** → 不顺 → 第二阶段：建SOP + 扩展智能体
3. **能自己跑了吗？** → 不能 → 第三阶段：内容生产力 + 持续迭代机制

## 优先级法则

```
P0: 解决最痛的（FAQ助手，降新人门槛）
P1: 拓展能力的（训练助手，加速上手）
P2: 锦上添花的（内容助手，提升生产力）
```

## 最小行动原则

> "保证最低行动不断线"

- 不追求大跃进，每周2个30分钟即可
- 任务在每次会议后明确拆解
- 卡住了先回到底座——底座不对，后面都白搭

## 行业定制 vs 通用方案

| 通用方案 | 行业定制 |
|---|---|
| 六间房通用模板 | 四模块法（产品/FAQ/案例/SOP） |
| 通用Prompt | 行业专属Skill |
| 标准交付节奏 | 根据行业淡旺季调整 |
