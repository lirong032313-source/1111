> **类型**：方法论卡 · 蓉蓉子原创  
> **归属**：02_我会什么  
> **关联母题**：M3-AI智囊团 / M2-小而美产品  
> **AI调用场景**：教学员搭 AI 团队 / 数字员工设计 / 直播 3 数字员工讲解

## 一句话定义

**不要把所有事喂给一个智能体——把"一人公司"按职能拆 5 个 AI 教练，每个对接一个知识库模块**。

---

## 团队架构

```
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│  商业教练    │  │  产品教练    │  │  内容教练    │
│  做决策/方向  │  │  设计产品    │  │  输出内容    │
└─────────────┘  └─────────────┘  └─────────────┘

┌─────────────┐  ┌─────────────┐
│  人生教练    │  │  销售教练    │
│  提升心力    │  │  丝滑成交    │
└─────────────┘  └─────────────┘
```

每个教练 = 一个独立智能体 + 对应的知识库模块

---

## 为什么不用单一大模型

❌ **单一大模型**：
- 喂太多→注意力分散，回答混乱
- 没有职能边界，输出"全能但不专精"
- 风格漂移严重

✅ **多智能体分工**：
- 每个智能体只读自己负责的房间
- 边界清晰，输出更专业
- 风格统一（受 CLAUDE.md 约束）

## 蓉蓉子实战案例（33 人实操营复盘）

她在 3 月那期实操营造了 **6 个 AI 分身**，干了 90% 的活：

| 分身 | 职能 | 节省时间 |
|---|---|---|
| 课程设计师 | 直播大纲 + PPT | 4 小时 |
| 预设问题库 | FAQ / 答疑 | 20 小时 |
| 话术生成器 | 社群公告 / 私聊 | 15 小时 |
| 作业批改官 | 评分 + 个性化反馈 | 8 小时 |
| 证书生成器 | 结营证书批量生成 | 2 小时 |
| 课程迭代官 | 提取共性痛点 → 改课程 | 5 小时 |

→ 总计省 **54 小时**，全用于"听见学员"

---

## 落地路径

1. **Phase 1**：先搭好六间房知识库（地基）
2. **Phase 2**：用 Claude Code / WorkBuddy 创建 5 个智能体（每个一个 SKILL.md）
3. **Phase 3**：每个智能体只授权读对应房间
4. **Phase 4**：跑通后微信里直接 @ 对应教练
