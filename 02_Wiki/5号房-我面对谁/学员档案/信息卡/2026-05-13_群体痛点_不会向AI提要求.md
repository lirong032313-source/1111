# 📋 群体痛点观察：不会向 AI 提要求

> 来源：2026-05-13 实操营群学员反馈
> 类型：🟠 共性问题（多数学员可能存在）
> 状态：🔍 待进一步收集 / 待设计解决方案

---

## 痛点概述

| 维度 | 内容 |
|------|------|
| **触发场景** | 实操营学习过程中，六间房搭好了内容之后 |
| **核心表达** | "房间都搞了内容，但感觉具体内容和自己要的还相差很远。还不知道怎么向AI提要求" |
| **问题本质** | 提示词工程（Prompt Engineering）能力缺失 + 对 AI 交互模式不熟悉 + **知识库内容与个人需求之间的"最后一公里"断层** |

## 📝 学员原话（Goldmine）

> "房间都搞了内容，但感觉具体内容和自己要的还相差很远。还不知道怎么向AI提要求"
> —— 实操营群学员，2026-05-13

### 原话拆解

| 原话片段 | 隐含信息 | 重要度 |
|---------|---------|--------|
| 「房间都搞了内容」 | 学员已经完成了知识库搭建动作（六间房结构有了、内容填进去了） | ⭐⭐⭐ 说明不是懒，是认真做了 |
| 「和自己要的还相差很远」 | **核心痛点：有骨架无灵魂**——内容是填进去了，但调用时AI给的不是自己要的 | ⭐⭐⭐⭐⭐ 这是真问题 |
| 「还不知道怎么向AI提要求」 | 表面是提示词问题，深层是**不知道怎么让已有的知识库为自己服务** | ⭐⭐⭐⭐ |

## 拆解分析

### 学员可能的具体表现

| 层面 | 可能的行为/困惑 | 隐含需求 |
|------|----------------|---------|
| **输入层** | 不知道该说什么、说多少、用什么结构 | 一套简单的提问框架/模板 |
| **预期层** | 不知道 AI 能做到什么程度，期望值模糊 | AI 能力边界说明 + 示例 |
| **迭代层** | 得到不满意的结果后不会追问/调整 | 迭代对话的方法论 |
| **场景层** | 不同任务（写文案/整理资料/分析问题）不会切换问法 | 场景化提示词库 |

### 根因判断（已验证 — 基于学员原话）

```
┌─────────────────────────────────────────────────────┐
│  表象："不知道怎么提要求"                              │
├─────────────────────────────────────────────────────┤
│  第一层：知识库有内容，但"不对口"                       │
│    → 房间搭了、内容填了，但调用时 AI 给的不是想要的     │
│    → 缺少"从个人需求→调取知识库→精准输出"的链路        │
├─────────────────────────────────────────────────────┤
│  第二层：没理解 AI 的"对话式协作"范式                    │
│    → 还在用搜索引擎思维（输关键词→得结果）              │
│    → 而不是对话思维（描述需求→迭代优化）               │
├─────────────────────────────────────────────────────┤
│  第三层：对自己要什么不够清晰                          │
│    → "和自己要的还相差很远" = 需求本身没结构化          │
│    → 这是产品思维/结构化表达的问题                     │
└─────────────────────────────────────────────────────┘

💡 关键洞察：
  学员的问题不是"没做"，而是"做了但没用起来"
  这比"不知道怎么开始"更隐蔽，也更需要关注
```
