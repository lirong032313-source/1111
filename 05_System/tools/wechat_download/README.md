# 🦞 公众号历史文章批量下载工具

> 一次性把你公众号的全部历史文章下载到 ob大脑 的 `01_Raw/公众号历史文章/`，作为 03_我说过什么的风格采样来源。

---

## 📋 这个工具能干什么

| 能力 | 说明 |
|---|---|
| ✅ 批量下载已发表文章 | 一次性拉全部，自动分页、自动去重 |
| ✅ 转 Markdown 格式 | 保留段落、标题、列表、链接、图片，去掉公众号样式干扰 |
| ✅ 元数据 frontmatter | 每篇带 title / url / 发布日期 / 摘要，方便 AI 后续处理 |
| ✅ 文件名带日期前缀 | `2026-03-18_坦白局：这场33人的实操营.md` 这种格式，时间排序天然 |
| ✅ 失败自动重试 | 网络抖动不会整批阵亡 |
| ✅ 风控延迟保护 | 默认每篇间隔 3 秒，避免触发微信反爬 |

## 🚀 三步走

### 第 1 步：装依赖（30 秒）

打开 Mac 的「终端」（Terminal），cd 到本工具目录：

```bash
cd "/Users/rong/Desktop/ob大脑/05_System/tools/wechat_download"
pip3 install -r requirements.txt --break-system-packages
```

> 💡 `--break-system-packages` 是 Mac 新版 Python 必加的参数。如果报错就去掉这个参数试试。

---

### 第 2 步：拿到你的 cookie 和 token（5 分钟）⭐ 关键步骤

#### 2.1 登录公众号后台

用 **Chrome 浏览器**（推荐）打开 [https://mp.weixin.qq.com](https://mp.weixin.qq.com)，扫码登录你的公众号。

登录后浏览器地址栏会变成类似：
```
https://mp.weixin.qq.com/cgi-bin/home?t=home/index&token=1234567890&lang=zh_CN
```

#### 2.2 复制 token

在地址栏 URL 里找 `token=` 后面那串数字（一般是 9-10 位）：

```
                                    ↓ 这串数字就是 token
https://mp.weixin.qq.com/.../home?t=home/index&token=1234567890&lang=zh_CN
```

把它复制下来。

#### 2.3 复制 cookie

1. 在公众号后台主页按 `F12`（或右键 → 检查），打开**开发者工具**
2. 切到 **Network**（网络）标签
3. 刷新页面（`⌘ + R`）
4. 在 Network 列表里随便点一个 `mp.weixin.qq.com` 的请求（最上面那个 home 请求即可）
5. 右侧面板找 **Headers** → **Request Headers** → **Cookie:** 这一行
6. **整段**复制 `Cookie:` 后面的内容（很长，一般几百字符，不要漏）

#### 2.4 填到脚本里

用文本编辑器打开 `download_wechat.py`，找到顶部的 CONFIG 区域：

```python
CONFIG = {
    "cookie": "粘贴你的完整 cookie 字符串到这里",   # ← 替换这行
    "token": "粘贴你的 token 数字到这里",            # ← 替换这行
    ...
}
```

把刚才复制的 cookie 和 token 粘贴进去（**保留双引号**）。

> ⚠️ **重要**：cookie 和 token **不要外发给任何人**，它们等同于你的公众号登录态。本脚本只在你 Mac 本地运行，不会上传任何地方。

## ⚠️ 常见问题

### Q1: 报错"列表接口 HTTP 401" / "cookie 失效"

→ cookie 可能过期了（每次登录有效期约 2-4 小时）。重新打开公众号后台，按 2.3 步骤再复制一次 cookie。

### Q2: 报错"接口错误：freq control"

→ 触发了微信风控。把 `delay_seconds` 改到 5 或 10 秒，等 30 分钟后再跑：

```python
"delay_seconds": 8,
```

### Q3: 部分文章下载失败"正文区域未识别"

→ 个别文章用了非常规模板（如纯图片图文）。这些会被跳过，最后报告里会显示失败数。可以从 `_article_list.json` 找到这些文章的 url 单独处理。

### Q4: 我想只下某段时间的文章

→ 现在脚本是按时间倒序拉，所以你可以用 `limit` 控制只拿最近 N 篇：
```python
"limit": 50,   # 只下最近 50 篇
```

如果想下"2024 年那批"，可以下完所有然后筛选——文件名前缀就是日期。

### Q5: 图片会被下载吗？

→ 图片以**链接形式**保留在 Markdown 里（不会本地化）。原因：
- 图片体积大，154 篇 × 平均 5 张图 = 几 GB
- AI 在做风格采样时不需要图片（只看文字）
- 如果你想保留图片，告诉我，我加一个 `download_images: True` 选项
