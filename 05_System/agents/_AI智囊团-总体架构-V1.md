# 蓉蓉子 AI 智囊团｜总体架构 & 实现路径 V1.0

> **本文件性质**：一人公司 AI 数字组织的**顶层设计**，所有分身的角色定义、技术栈、实施节奏都从这里展开。
> **创建时间**：2026-05-13
> **维护节奏**：每月迭代一次，记录实际跑通后的调整。
> **核心原则**：不堆工具，搭组织——把"自己一个人"扩展成"一个 AI 团队"。

---

## 一、为什么要做这件事（北极星目标）

**一句话目标**：把蓉蓉子从"一个一直在燃烧的人"，扩展成"一个 7×24 在线、可扩展的小型 AI 组织"。

**三个验证标志**：
1. 内容产出量提升 3-5 倍（朋友圈/公众号/小红书无须每篇从零开始）
2. 1v1 咨询前的"诊断+方案准备"时间，从 2 小时降到 30 分钟
3. 用户在非工作时间也能得到有质量的回复

**底层信念**：你已经有了别人没有的最大资产——**结构化的六间房知识库**。剩下的事是把它从"档案"激活成"组织"。

---

## 二、组织架构（三层模型）

### L1 · 知识库底座（共享层）
所有分身共用同一份 `02_Wiki/` 知识库。这是最关键的设计——**一份资产，多处复用**。
分身的差异不在于"知道什么"，而在于"被叫来做什么角色"。

### L2 · AI 分身层（角色层）
四个分身，分两类：

| 分身 | 类别 | 服务对象 | 核心使命 |
|---|---|---|---|
| 📝 内容分身 | 对内 | 蓉蓉子本人 | 内容生产（公众号/小红书/朋友圈/视频） |
| 🎯 产品咨询分身 | 对内 | 蓉蓉子本人 | 用户诊断 + 产品方案设计 |
| 🧠 商业智囊分身 | 对内 | 蓉蓉子本人 | 商业决策 + 复盘 + 目标拆解 |
| ✨ AI 蓉蓉子 | 对外 | 用户 | 答疑 + 引流 + 留资 |

### L3 · 可视化入口层（界面层）
**对内**：一个统一的"智囊团控制台"，让你能像打开 ChatGPT 一样，左侧选分身、右侧聊天，对话历史归档。
**对外**：一个独立的 H5 页面（首期），后期可演化成小程序。

## 三、四个分身的详细设计

### 📝 分身 1｜内容分身

| 维度 | 定义 |
|---|---|
| **角色定位** | 你的"AI 内容编辑部"——既会写、又懂你的风格 |
| **使用频率** | ⭐⭐⭐⭐⭐（最高频，每天可能用 3-5 次） |
| **知识库调用优先级** | 3 号房（风格） > 1 号房（故事） > 6 号房（观点） > 5 号房（用户语言） |
| **核心技能（4-6 项）** | ① 公众号长文起草 ② 小红书图文（含 emoji 节奏） ③ 朋友圈短文 ④ 视频脚本 ⑤ 标题 10 选 1 ⑥ 从素材中提取金句 |
| **典型输入** | 一句选题 / 一段语音转写 / 一篇灵感碎片 / 一个新观点 |
| **典型输出** | 直接可发布或 80% 完成的文稿 |
| **工作流** | 选题 → 调用 5 号房（找用户痛点）→ 调用 3 号房（套蓉蓉子节奏）→ 出大纲 → 出初稿 → 风格化润色 → 配图建议 |
| **优先级** | 🥇 第一个搭（高频高价值，立刻减负） |

---

### 🎯 分身 2｜产品咨询分身

| 维度 | 定义 |
|---|---|
| **角色定位** | 你的"AI 产品军师"——咨询前后、方案输出的得力助手 |
| **使用频率** | ⭐⭐⭐⭐（每周用 5-10 次，每次咨询前/后） |
| **知识库调用优先级** | 4 号房（产品矩阵） > 5 号房（客户画像） > 6 号房（决策框架） > 案例库 |
| **核心技能** | ① 用户初诊（基于聊天记录提炼现状/卡点） ② 推荐合适的产品档位 ③ 输出 1v1 咨询大纲 ④ 起草咨询报告初稿 ⑤ 定价策略测算 ⑥ MVP SOP 生成 |
| **典型输入** | 用户咨询前的聊天记录 / 一段语音咨询的逐字稿 / 一个产品想法 |
| **典型输出** | 结构化诊断报告 / 方案 PPT 大纲 / 给用户的回复初稿 |
| **工作流** | 输入用户素材 → 用 5 号房做画像分类 → 用 4 号房匹配合适产品 → 用 6 号房做决策评估 → 输出三件套（诊断+方案+下一步） |
| **优先级** | 🥈 第二个搭（直接节省专业时间） |

---

### 🧠 分身 3｜商业智囊分身

| 维度 | 定义 |
|---|---|
| **角色定位** | 你的"AI 商业教练"——重要决策前的第二大脑 |
| **使用频率** | ⭐⭐⭐（每周用 2-3 次，做重大决策时） |
| **知识库调用优先级** | 6 号房（决策框架） > 4 号房（商业模型） > 案例库 > 1 号房（价值观） |
| **核心技能** | ① SWOT/优先级矩阵分析 ② 月度/季度复盘 ③ 目标拆解（OKR/KPI） ④ 商业模型画布 ⑤ 风险预案 ⑥ 同类案例对照 |
| **典型输入** | 一个困扰你的商业问题 / 一个待决策的选项 |
| **典型输出** | 多角度分析 + 推荐方案 + 风险提示 + 行动清单 |
| **工作流** | 输入问题 → 调用 6 号房决策原则 → 调用 4 号房商业现状 → 多角度分析 → 给出 2-3 个方案 → 推荐其中一个并给理由 |
| **优先级** | 🥉 第三个搭（低频但高价值，可以晚一点） |

## 四、四阶段实施路径（共 3-4 个月）

### 🚀 Phase 1｜底座完善（第 1-2 周）

**目标**：让知识库真正"AI 可调用"，不是简单的 Markdown 堆。

**关键动作**：
1. 给每个 .md 文件加 frontmatter 元数据（`房间号`、`类型`、`关键词`、`更新日期`）
2. 把信息卡的颗粒度统一（每张信息卡 ≤ 500 字，结构化字段）
3. 把 `01_Raw/咨询逐字稿/` 中已脱敏的核心片段，整理 30-50 段进 3 号房作为"风格采样"
4. 在每个房间根目录写一个 `index.md`，作为该房间的 TOC + 导航

**产出**：可被任何 RAG 系统稳定检索的知识库 v2.0

**成本**：你自己 8-12 小时（或者花 ¥500-800 让人帮你打 frontmatter）

---

### 🛠 Phase 2｜单分身 MVP（第 3-6 周）

**目标**：先把**内容分身**跑通，用 2-4 周时间验证"分身真的能减负"。

**关键动作**：
1. 选平台：**推荐 Dify**（开源 + 云版本均可，支持工作流编排，比 Coze 更灵活）
2. 把 `02_Wiki/` 上传到 Dify 知识库
3. 写 4-6 个"应用"：公众号长文应用、小红书应用、朋友圈应用、起标题应用
4. 每个应用有自己的 system prompt（参考 AI 分身-V1 的写法）
5. 内部使用 2 周，每天记录"哪些好用、哪些不够"

**产出**：一个内容分身 + 4-6 个具体应用

**成本**：Dify 云版本约 ¥299-999/月，自己投入 20-30 小时调试

**判断成功的标准**：你写公众号的时间，从 3 小时降到 1 小时

---

### 🔧 Phase 3｜多分身扩展（第 7-12 周）

**目标**：搭起产品咨询分身 + 商业智囊分身，开始有"AI 团队"的感觉。

**关键动作**：
1. 复用 Phase 2 的搭建模式，分别建产品咨询分身、商业智囊分身的工作流
2. 设计**分身间协作**：比如内容分身写完一篇文章，可以一键推给产品咨询分身做"产品落地建议"
3. 建立**对话日志归档**：每个分身的对话自动存到 Notion 或 `03_Dashboard/` 里
4. 开始观察"哪些经验应该回流到知识库"——这是分身让知识库变厚的关键

**产出**：3 个对内分身全部上线 + 一套协作机制

**成本**：自己投入 30-40 小时

---

### 🎨 Phase 4｜可视化 + 对外发布（第 13-16 周）

**目标**：内部用熟之后，再把"AI 蓉蓉子"对外发布。

**关键动作**：
1. **对内控制台**：用 Notion 嵌入 Dify 应用的 iframe，做一个"智囊团首页"
   - 顶部：四张分身卡片
   - 中间：当前活跃对话
   - 底部：最近 7 天产出（文章数 / 咨询数 / 决策数）
2. **对外 H5**：直接用 Dify 提供的 web app（不需要前端开发），自定义品牌色和欢迎语
3. **入口铺设**：公众号自动回复 / 朋友圈分享 / 小红书简介挂链
4. **数据观察**：每周看 Top10 用户问题 → 反向迭代 Prompt 和知识库

**产出**：完整可对外的 AI 蓉蓉子 H5

**成本**：约 ¥0-2000（Notion + Dify 月费），自己投入 15-20 小时

## 五、技术栈选型（推荐组合）

| 层级 | 推荐方案 | 替代方案 | 理由 |
|---|---|---|---|
| **知识库存储** | 仍用 Obsidian / 飞书文档 | Notion | 已有的不要换 |
| **RAG + Agent 引擎** | **Dify**（开源/云版） | FastGPT / Coze | Dify 支持工作流编排 + 多模型，最适合多分身架构 |
| **底层模型** | DeepSeek（性价比） + Claude（写作类） | GPT-4 / 通义千问 | 内容分身建议 Claude，其余 DeepSeek 即可 |
| **向量数据库** | Dify 内置 | Qdrant / Chroma | 不用自己搞 |
| **对内可视化** | Notion 嵌入 | Obsidian Dataview | 最低成本上线 |
| **对外 H5** | Dify 自带 web app | 自研 Next.js | 第一阶段用自带的就够 |
| **数据归档** | Notion 数据库 | Airtable | 看你已有工具习惯 |

**月度运营成本估算**（Phase 4 全跑通后）：
- Dify 云版：¥299-999
- Claude API：¥200-500（按 100 万 tokens 算）
- DeepSeek API：¥50-200
- **合计**：¥600-1700/月（一个普通员工 1 天的成本）

## 七、关键风险与应对

| 风险 | 严重度 | 应对方案 |
|---|---|---|
| 知识库不够"AI 可读"，分身回答质量差 | 🔴 高 | Phase 1 必须做透，不省 |
| 同时搭 4 个分身导致精力分散 | 🔴 高 | 严格按顺序：内容→产品→商业→对外 |
| 对外分身上线后回答出错被截图 | 🟡 中 | 强制 V1 边界规则，每周抽查日志 |
| 模型 API 涨价 / 平台跑路 | 🟡 中 | Dify 开源可自部署，知识库始终在自己手里 |
| 分身越搭越复杂、自己用不起来 | 🟠 中低 | 每个分身只做 4-6 个核心技能，不堆功能 |

## 九、现在最该动手的 3 件事

1. **决定平台**：Dify 还是 FastGPT？（推荐 Dify，但你自己确认）
2. **做 Phase 1 的元数据规范**：用 1 个周末把所有 .md 加 frontmatter
3. **写内容分身的 system prompt**：参考 AI 分身-V1 的格式，但聚焦"内容生产"场景

我可以帮你做的下一步：
- ① 写**内容分身 V1 Prompt**（最高优先级，影响日常产出）
- ② 起草**知识库 frontmatter 标准 + 模板脚本**（影响所有分身的回答质量）
- ③ 出**Dify vs FastGPT vs Coze 的详细对比表**（帮你决策）
