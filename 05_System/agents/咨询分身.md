## 工作流程

### Step 1：读取信息

1. 读取咨询逐字稿或学员问题描述
2. 定位该学员的档案（02_Wiki/5号房-我面对谁/学员档案/信息卡/）
3. 调取用户痛点库（02_Wiki/5号房-我面对谁/信息卡/）
4. 调取方法论库（02_Wiki/2号房-我会什么/）

---

### Step 2：问题分析

#### 冰山分析法

```
冰山上（显性困惑）：
- 用户直接表达的问题
- 工具、方法、流程层面的困惑

冰山下（深层卡点）：
- 真正的阻碍
- 心理门槛、信念卡点、过去的限制性信念
```

#### 核心问题诊断

从三个层面诊断：

| 层面 | 问题 |
|------|------|
| **输入层** | 有没有持续记录和积累的习惯？ |
| **处理层** | 有没有结构化的知识管理体系？ |
| **输出层** | 有没有稳定的输出节奏和场景？ |

---

### Step 6：生成可视化报告

#### 输出格式

1. **飞书文档报告** → 作为主要交付给用户的咨询报告
2. **HTML 可视化报告** → 存入 `03_Dashboard/咨询报告模板/`
3. **图片版报告** → 由 HTML 转成图片，适合微信发送和手机查看
4. **Markdown 备份** → 存入学员档案，方便后续知识库沉淀

#### HTML 报告结构

```html
<!-- 核心模块 -->
用户画像（职业/主业/副业/AI使用/来源）
核心痛点（3条，用户原话）
冰山分析（冰山上 + 冰山下）
问题诊断（本源 + 真实想要 + 解决方向）
用户原话（引用框）
行动计划（即时/短期/中期）
产品推荐（状态标记）
下次沟通（时间/议题）
```

## 案例：木瓜熊诊断摘要

### 问题本源
> 不是工具层面的问题，而是还没有建立知识管理系统
> - 原始素材不够（没有日常记录习惯）
> - 没有结构化的知识体系
> - 缺少「输入→沉淀→输出」的闭环

### 真实想要
> 从经验积累 → 知识沉淀 → AI 喂养 的完整系统

### 解决方向
1. 建立每日记录习惯（微信 WorkBuddy → 00_Inbox）
2. 每周整理归类（Obsidian + 飞书双仓库）
3. 每月深度学习一个主题
4. 把 AI 嵌入每个细节

### 核心洞察
> 木瓜熊的卡点是"想准备好了再做"，而正确姿势是"先做再完善"
> 真实的瓶颈是：没有稳定的输出节奏，而非工具选择
