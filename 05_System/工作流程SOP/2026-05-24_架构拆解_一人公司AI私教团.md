## 一、先回答一个问题：为什么是"私教团"而不是"用更多AI工具"？

### 大多数人踩的坑

```
用Cursor写代码 → 不满意 → 换Windsurf
用Coze搭智能体 → 太复杂 → 换Dify
用ChatGPT写文章 → 不像自己 → 换Claude
```

每一次换工具，都是"从头开始"——配置重写、提示词重调、知识库重建。

### 蓉蓉子的答案

> **不是换工具。是搭组织。**

一个人 ≠ 只能做一个人的事。一个人 + 一座军火库（六间房） = 能调度一支 AI 团队。

### 核心设计原则

| 原则 | 说明 |
|---|---|
| **底座 > 工具** | 六间房搭好，换什么工具都只改调用方式 |
| **角色 > 功能** | 不是"我有一个写作功能"，是"我有小蜜这个内容分身" |
| **真实 > 完美** | 分身从你的真实知识库里长出来，不是角色扮演 |
| **先跑通，再复制** | 自己用 30 天验证，再抽象成学员模板 |

## 二、架构演变：从理想到现实

### V1（5/13 规划版）→ V2（5/18 实战版）→ 当前（5/24 直播版）

| 版本 | 角色数 | 角色 | 变化原因 |
|---|---|---|---|
| **V1 智囊团** | 4 | 内容/产品咨询/商业智囊/对外蓉蓉子 | 概念验证，先想清楚"几个分身" |
| **V2 私教团** | 5 | 小蜜/小李/石榴/小懂/小策 | 加入"交付助手"和"产品教练"，改名让角色更人格化 |
| **当前直播版** | 5 | 🐾石榴/📝小蜜/🎯小策/📂小勤/💰钱老 | 实际使用时发现：小李=小策合并、小懂→小勤、石榴升级为首席 |

### 关键洞察

> **不是你设计谁，是跑着跑着它们自己长出来的。**

小蜜是从 3 号房（我会说什么）长出来的——因为有风格采样和金句库，她自然就会写。
钱老是从 4 号房（我卖什么）长出来的——因为有产品矩阵和定价模型，他自然就能算。
石榴是从全局长出来的——因为她读所有房间，自然就成了首席。

## 三、当前团队：谁是谁，负责什么

```
              ┌─────────────────┐
              │   你（CEO）     │
              │  做决策、把方向  │
              └────────┬────────┘
       ┌───────────────┼───────────────┐
       ↓               ↓               ↓
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 🐾 石榴   │    │ 📝 小蜜   │    │ 🎯 小策   │
│ 首席搭档  │    │ 内容分身  │    │ 产品分身  │
│ 战略+执行 │    │ 选题+写作  │    │ 设计+迭代  │
└──────────┘    └──────────┘    └──────────┘
       ↓               ↓               ↓
┌──────────┐    ┌──────────┐
│ 📂 小勤   │    │ 💰 钱老   │
│ 执行分身  │    │ 商业分身  │
│ 整理+排期 │    │ 定价+转化  │
└──────────┘    └──────────┘
```

### 每个分身详细拆解

| 分身 | 调用房间 | 核心技能 | 典型输入 | 典型输出 | 频率 |
|---|---|---|---|---|---|
| 🐾 **石榴** | 全部 | 战略分析、任务调度、日报周报、入库协调 | "帮我分析下这个决策" | 多角度分析+行动建议 | 每天 |
| 📝 **小蜜** | 3号房>1号房>5号房 | 选题、公众号、小红书、润色、起标题 | 碎片/灵感/一句话 | 选题池→大纲→初稿 | 每天3-5次 |
| 🎯 **小策** | 4号房>2号房>6号房 | 产品设计、SOP打磨、产品迭代、复盘 | "帮我优化这个产品" | 产品诊断+优化方案 | 每周2-3次 |
| 📂 **小勤** | 2号房>全部 | 资料整理、入库归类、排期管理 | "帮我把这个入库" | 结构化信息卡+索引更新 | 每天 |
| 💰 **钱老** | 4号房>6号房 | 定价策略、商业模式、转化分析 | "这个产品怎么定价" | 定价方案+竞争分析 | 每周1-2次 |

### 最重要的设计：一份资产，5 个视角

所有分身读取同一套六间房，但每个只调用自己需要的房间：

```
         ┌─────────────────────────┐
         │    六间房知识库          │
         │  01  02  03  04  05  06 │
         └──┬───┬───┬───┬───┬───┬──┘
            │   │   │   │   │   │
      ┌─────┘   │   │   │   │   └─────┐
      ↓         ↓   ↓   ↓   ↓         ↓
    石榴      小蜜 小策 小勤 钱老     ...
   （全读）  （3,1,5）（4,2,6）（2,全）（4,6）
```

> **教学要点**：你不是搭 5 个独立的 AI。你是搭 1 个底座，然后长出 5 双不同用途的眼睛。

## 四、搭建过程：分 4 步，每步 1 周

### Step 1：先搭一个房间——3 号房（我说过什么）

这是所有分身的起点。

**为什么从 3 号房开始？**
- 3 号房有你的金句和风格采样 → 小蜜立刻能用
- 学员最快能感受到"它好像在懂我"
- 10 篇文章、50 条朋友圈就够了

**产出**：小蜜的第一个版本——能出选题、能写初稿（语气有 60-70% 像你）

---

### Step 2：补 1 号房 + 5 号房——让小蜜更像你

- 1 号房（我是谁）：身份、故事、价值观 → 初稿不再是"正确的废话"
- 5 号房（我面对谁）：用户痛点、典型场景 → 选题能戳中读者

**产出**：小蜜 v2——语气像、选题准、能调用你的真实故事

---

### Step 3：激活 4 号房——钱老和小策出场

- 4 号房（我卖什么）：产品矩阵、定价策略、交付流程
- 钱老从 4 号房长出来 → 能给你定价建议
- 小策从 4+2+6 号房长出来 → 能帮你迭代产品

**产出**：钱老 + 小策的第一个版本

---

### Step 4：全局打通——石榴 + 小勤收尾

- 石榴从全部房间长出来 → 能做全局调度和战略分析
- 小勤从 2 号房长出来 → 能帮你整理、入库、排期

**产出**：完整 5 人团队 v1，30 天自用验证

### 每阶段的验证标准

| 阶段 | 你问自己 |
|---|---|
| W1 小蜜v1 | "它写的选题，我能挑出一个想要的吗？" |
| W2 小蜜v2 | "朋友读完，能看出这是我写的吗？" |
| W3 钱老+小策 | "它的定价建议，我认真考虑了吗？" |
| W4 石榴+小勤 | "我少了 30% 的重复劳动吗？" |

## 六、今晚你可以带走的东西

### 今晚就搭一个——小蜜（内容分身）

**最小启动清单**：

```
□ 准备 10 篇你写过的文章 / 50 条朋友圈
□ 放到一个文件夹里（就叫"3号房-我说过什么"）
□ 写一句 CLAUDE.md 开头：
    "我是 [你的名字]，[一句话介绍]。
     我的写作风格：________"
□ 跟 AI 说："读一下这些内容，然后用我的语气写一篇关于 XX 的文章"
```

不是搭六间房。先搭一间。

### 怎么套用到你自己的业务

我的 5 个分身 → 你的 N 个分身：

| 我 | 你 |
|---|---|
| 小蜜（内容） | 你的内容分身（公众号/小红书/视频脚本） |
| 小策（产品） | 你的产品分身（课程设计/服务迭代） |
| 钱老（商业） | 你的商业分身（定价/渠道/转化） |
| 小勤（执行） | 你的执行分身（排期/入库/提醒） |
| 石榴（首席） | 你的首席搭档（全局调度） |

**核心不变**：先搭底座（你的知识库），再长分身（你的业务场景）。

## 七、常见问题

| Q | A |
|---|---|
| 我没有 6 个房间怎么办？ | 先搭 1 个。3 号房最简单——10 篇文章就行 |
| 分身的回答不准怎么办？ | 不是改 Prompt，是补知识库。分身不准=底座缺内容 |
| 用哪个工具？Dify？Coze？ | 先别管工具。拆出你自己的 SOP，工具只是实现方式 |
| 搭完多久能见效？ | 第一个分身（小蜜）1 周就有感觉。完整团队 30 天 |
