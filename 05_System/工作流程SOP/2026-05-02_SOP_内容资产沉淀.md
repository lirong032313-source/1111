## 什么是内容资产沉淀

让你的每一篇创作都能"可复用"，而不是"写完就扔"。

**核心理念：好内容值得被重复调用10次，而不是写10篇新内容。**

---

## 沉淀SOP流程

```
产出内容
    │
    ▼ 判断价值
    ├─ 是否解决了一个问题？───→ ⬇ 是
    ├─ 是否有独特洞察？────→ ⬇ 是
    └─ 是否可复用？───────→ ⬇ 是
           │
           ▼ 提取"核"
           ├─ 核心观点（1句）
           ├─ 适用场景（3个）
           └─ 行动指引（1条）
           │
           ▼ 入库
           ├─ 金句库（3号房）
           ├─ 方法论卡（2号房）
           ├─ 案例库（案例库）
           └─ 选题池（3号房）

    └─ ⬆ 否 → 入草稿箱/待优化
```

## 入库规范

### 方法论卡模板

```markdown
# 方法论卡：XXX

> 来源：某篇文章 / 某次咨询 / 某个案例
> 适用场景：
> 核心要点：
> 使用步骤：
> 注意事项：
```

### 金句卡模板

```markdown
# 金句：XXX

> 适用语境：
> 可以用在：
> 背后理念：
```

### 案例卡模板

```markdown
# 案例：XXX

> 背景：
> 关键动作：
> 结果：
> 可复用点：
```

## 实际案例

**文章产出→方法论沉淀示例：**

| 产出 | 提取 | 沉淀位置 |
|------|------|----------|
| 《为什么你的知识库搭不起来》 | "底座比咒语重要" | 2号房-方法论卡 |
| 文中提到的3个坑 | "知识库3坑自查表" | 2号房-方法论卡 |
| "在轨道中寻找旷野" | 金句 | 3号房-金句库 |
| 快乐星球案例 | "3天MVP验证法" | 2号房-方法论卡 |

---

## 底层原则

1. **每一篇好内容都值得被抽出"干货"**
2. **不求多，求被调用率**
3. **入库是为了更好的产出**
