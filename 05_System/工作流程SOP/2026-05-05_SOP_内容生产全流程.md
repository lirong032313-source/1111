# SOP：内容生产全流程

> 类型：工作流程SOP
> 位置：05_System/工作流程SOP/
> 覆盖场景：公众号写作 / 小红书写作 / 朋友圈扩写
> 入口渠道：微信ClawBot / WorkBuddy / 直接对话
> 更新时间：2026-05-05

---

## 一、多渠道入口

```
┌──────────────┐
│  微信ClawBot  │ ← 输入碎片/灵感/指令
└──────┬───────┘
       │
┌──────────────┐
│  WorkBuddy   │ ← 输入写作指令
└──────┬───────┘
       │
┌──────────────┐
│  直接对话     │ ← 输入明确主题
└──────┬───────┘
       │
       ▼
┌──────────────────────────────┐
│    石榴（此AI助手）          │
│  1. 意图识别                 │
│  2. 知识库调取               │
│  3. 内容生成                 │
│  4. 配图生成                 │
│  5. 发布（ask确认后）        │
└──────────────────────────────┘
```

**从微信ClawBot入口**：当前支持通过openclaw桥接，输入碎片或指令→触发WorkBuddy自动化→石榴处理。
**当前限制**：自动化任务暂不支持直接推送到微信，需通过OpenClaw桥接或ask确认后在WorkBuddy查看结果。
**待确认**：如需要在微信内完成确认和自动回复，需配置OpenClaw的对话链路。

## 二、意图识别规则

### 输入关键词 → 意图映射

| 输入关键词 | 意图 | 输出格式 |
|---|---|---|
| "写公众号" / "写一篇" / "公众号" | 公众号文章 | 2000-3000字 + 配图 |
| "写小红书" / "小红书" / "笔记" | 小红书笔记 | 800-1500字 + 配图 |
| "朋友圈扩写成公众号" / "朋友圈改公众号" | 朋友圈→公众号 | 2000-3000字 + 配图 |
| "朋友圈扩写成小红书" / "朋友圈改小红书" | 朋友圈→小红书 | 800-1500字 + 配图 |
| "扩写" + 平台名 | 任意输入→指定平台 | 对应平台格式 |
| "第②篇" / "第X篇" | 系列续写 | 按系列规划 |

### 无明确平台时的默认判断

```
用户输入的主题内容
    │
    ▼ 判断
    ├─ 提及"知识库/架构/方法论/底层" → 公众号深度文
    ├─ 提及"产品/工具/模板/实战"     → 公众号深度文
    ├─ 内容长度 < 200字              → 先问：公众号还是小红书？
    ├─ 内容像朋友圈/日常分享          → 先问：扩写成公众号还是小红书？
    └─ 内容像干货/方法论              → 默认公众号
```

## 三、内容创作模式

### 模式A：模糊主题 → 先出标题

**触发条件**：用户给的主题比较模糊（如"帮我写篇关于知识库的文章"）

```
用户给模糊主题
    ↓
石榴生成3个标题选项（15-20字）
    ↓
用户选择1个标题
    ↓
石榴按选中的标题写全文
    ↓
用户审阅 → 确认
    ↓
生成配图 → 发布
```

### 模式B：明确主题 → 直接写

**触发条件**：用户给了明确的标题或主题（如"写一篇《我把自己'拆解'给AI后》"）

```
用户给明确主题/标题
    ↓
石榴直接写全文
    ↓
标注需要润色的地方
    ↓
用户审阅 → 确认
    ↓
生成配图 → 发布
```

## 四、内容生产规范（公众号版）

### 4.1 标题规范
- 3个备选标题（仅模式A需要）
- 每个15-20字
- 包含：痛点/反常识/数字/对比
- 示例风格：
  - "为什么你的XX总是XX？"
  - "我把自己XX后，它终于XX了"
  - "XX的本质，不是XX，而是XX"

### 4.2 文章结构

```
① 开头钩子（100-200字）
   - 反直觉观点 / 扎心问题 / 场景共鸣
   
② 正文（3-4个小标题，每个300-500字）
   - 长段落形式，非列表
   - 逻辑递进：问题 → 认知 → 方法论 → 案例
   
③ 结尾升华（100-200字）
   - 呼应开头，完成闭环
   - 温暖有力，种草产品和价值观
```

### 4.3 写作风格清单

```
✅ 语气：温暖、随意但深刻，像朋友思想碰撞
✅ 人称：第一人称"我"
✅ 结构：长段落，不用列表
✅ 故事：基于真实经历，不编造
✅ 语言：直接、富有洞察力、有韵律感
✅ 调性：温暖有趣有价值

❌ 禁用词：不是...而是、我愣了一下、恍然大悟
❌ 禁用：机械行动指南、说教感、营销话术
❌ 禁用：正确的废话（"我们要努力"）
```

### 4.4 知识库调用顺序

```
① CLAUDE.md（了解你是谁、语气风格、禁忌）
② 02_Wiki/3号房-我说过什么/风格采样/（学习语气）
③ 02_Wiki/3号房-我说过什么/金句库/（插入金句）
④ 02_Wiki/2号房-我会什么/（方法论支撑）
⑤ 02_Wiki/5号房-我面对谁/（用户痛点校准）
⑥ 02_Wiki/1号房-我是谁/（故事/价值观引用）
```

### 4.5 需要标注的润色点

写完文章后，在文末标注：

```markdown
---
✏️ 需要蓉蓉子润色的地方：
1. [第X段] 故事细节：_____ 需补充真实信息
2. [第X段] 案例数据：_____ 需确认是否准确
3. [第X段] 语气是否偏AI：_____ 需调整
---
```

## 五、配图生成规范

### 5.1 何时配图

用户确认文章内容满意后，问：

> "需要我生成配图吗？包含1张头图 + X张内页插图。"

### 5.2 配图清单（公众号）

| 序号 | 位置 | 内容 | 说明 |
|---|---|---|---|
| 1 | 头图（封面） | 文章主题视觉化 | 900×500px裁切 |
| 2 | 开头插图 | 场景/痛点可视化 | 如："死库"的3个问题 |
| 3 | 中间插图 | 关键方法论/架构 | 如：三层架构图 |
| 4 | 结尾插图 | 升华/行动引导 | 如：一句话金句图 |

### 5.3 配图风格

```
赛博霓虹风：
- 背景：深蓝 #0D1B2A → 深紫 #1A1A2E
- 主色：霓虹粉 #FF00FF / 霓虹蓝 #00FFFF
- 辅色：温暖橙 #FF6B35 / 温暖黄 #FFD700
- 文字：白色 #FFFFFF，霓虹发光效果
- 装饰：几何图形 + 细线条 + 粒子光点
```

## 九、微信ClawBot（OpenClaw）路径说明

**基于OpenClaw配置，用户可以在微信中完成全部操作。**

### 完整路径

```
微信ClawBot ← 用户输入主题/碎片/指令
    ↓
OpenClaw桥接
    ↓
石榴处理（意图识别 → 调知识库 → 写文章 → 生成配图）
    ↓
推送到微信 ← 用户确认是否发布
    ↓
用户确认 → 调wechat-article-publisher发到公众号草稿箱
```

### 在微信中可以做的事
- ✅ 输入模糊主题 → 石榴出3个标题 → 用户在微信选
- ✅ 输入明确主题 → 石榴直接开写 → 用户在微信确认
- ✅ 发朋友圈原文 → 石榴扩写成公众号/小红书 → 用户在微信确认
- ✅ "写公众号" / "写小红书" → 石榴识别意图 → 按格式输出
- ✅ 确认后 → 石榴调wechat-article-publisher → 发到草稿箱
- ✅ 用户在公众号后台预览 → 手动发布
